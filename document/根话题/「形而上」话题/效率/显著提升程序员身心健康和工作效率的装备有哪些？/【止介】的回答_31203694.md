# 显著提升程序员身心健康和工作效率的装备有哪些？
- 点赞数：5413
- 更新时间：2016年12月26日14时41分51秒
- 回答url：https://www.zhihu.com/question/23165812/answer/31203694
<body>
 <p data-pid="ALXZad8R">曾经有很长的一段时间一直在学习和探索提升工作效率&amp;身心健康的方法。讲讲我个人的经历吧！</p>
 <p data-pid="fQNVj8_7"><b>工作效率：（</b><b>转载请注明来源）</b></p>
 <p data-pid="IXQB4ELV"><b>加速度</b>（一个电脑的有用程度和它里面装的软件数成反比）</p>
 <ul>
  <li data-pid="Sl_H67DP">给电脑换上SSD吧，你再也不用为了打开Photoshop而等待一分钟了，只需3秒！</li>
  <li data-pid="GPzNN4_p">给电脑加条内存吧，开虚拟机再也不用等待菊花了</li>
  <li data-pid="c7nJVdLU">IDE上哪些从来都不用的Plugin给Remove掉吧</li>
  <li data-pid="odKoDBDm">Chrome上打开chrome://memory-redirect/你会惊奇的发现一个再简单不过的Extension也得占用几十M内存，长时间不用的扩展还是删掉吧</li>
  <li data-pid="dnkeze9c">开机启动项赶紧T掉（Mac用户不在此列，我们都是几个月不关机的）</li>
  <li data-pid="EgLIKTir">舒适的网络环境绝对是非常重要的，还在用小水管，赶紧去换成10M光纤吧，体验下网上冲浪的感觉！</li>
  <li data-pid="gScNYN8M">多显示器绝壁是攻城师的不二利器啊！左边敲代码，右边预览、左边敲代码，右边LOG、左边敲代码，右边看电视</li>
  <li data-pid="8BXo4LzI">实在只有一个显示器也行啊，多建几个虚拟桌面切换</li>
  <li data-pid="jxL7-9od">定制好的你的桌面图标（Mac Dock图标），把常用的提上去</li>
  <li data-pid="UwPR-Vk0">使用Quicksilver、Alfred之类的软件能有效的提高操作效率</li>
  <li data-pid="9OIR2ehj">键盘的速度绝对比鼠标快，所以还是多记一记快捷键吧。Chrome下的Vimium和Firefox下面的Vimperator上手后对比下用数据操作的时间你会合不拢嘴的</li>
 </ul>
 <p data-pid="__qX8a9T"><b>专注</b></p>
 <ul>
  <li data-pid="jDpOAMTm">Coding的时候关掉手机提醒吧</li>
  <li data-pid="Fxpbrwho">邮件通知、QQ消息等简直是让人分散注意力的罪魁祸首</li>
  <li data-pid="m6gt3DBv">合理的挑选工作时间也很重要（当然得条件允许了,我是感觉每天晚上12点后的三个小时效率非常高，原谅我每天3点睡觉）</li>
  <li data-pid="nKDEc0fI">一杯茶或咖啡在快没精神的时候是很有作用的</li>
  <li data-pid="c1jOYEkV">你机箱是多少天没清理了，风扇的声音都快盖过拖拉机了</li>
  <li data-pid="AcGl89mZ">对于有些人来说适当的音乐也能够使其更专注（我是那种要听着经典电视更能专注起来的，现在都能背下水浒传了）</li>
  <li data-pid="KLCyc91Z">我在做事的时候能不能不要扫我桌子底下</li>
  <li data-pid="zFzmIxby">哇擦，IDE里这个文件有23层目录，用鼠标去点导航不得点死啊！搜索优于导航,用快捷键进行全局查找,赶紧抛弃文件树吧</li>
  <li data-pid="YSlQBBW9">find、sed、grep、awk好用到流泪</li>
 </ul>
 <p data-pid="_VcCBssk"><b>自动化</b></p>
 <ul>
  <li data-pid="VoA-sJI2">很多时候我们都喜欢重复造轮子，但是不要造过一次还去造第二次好吗？</li>
  <li data-pid="lrLbZzLq">每次打开收藏夹里面一堆网站挨个浏览真的比RSS好吗？</li>
  <li data-pid="v15J3uBI">每次的重复操作虽然只是占用很少时间，但长时间这样也是很费时的。搞成自动化不好吗？（每周耗时超过1小时的都建议做成自动化）</li>
  <li data-pid="Y0LWmBho">搞自动化前好好衡量下值不值得，别给耗牛剪毛，越剪越多</li>
  <li data-pid="Pz9Rg8di">经常Google一些函数用法，赶紧将文档下下来本地查！（Mac下非常推荐Dash）</li>
 </ul>
 <p data-pid="_0Q7tOvy"><b>规范性</b></p>
 <ul>
  <li data-pid="64u4aO2b">这个小改动这样改就好了，没必要考虑那么多！- 结果后面又接二连三的来改这个或者这个引起的那个…改的时候考虑全面点</li>
  <li data-pid="_fVXN2sG">用上Git、Subversion之类的版本控制工具吧，这次出问题了马上回滚到上次正常版本</li>
  <li data-pid="omD0O49X">一定要写注释啊，别面下次看代码又从头看起！（被后面接手的人骂肯定不是滋味）</li>
  <li data-pid="NHR1rENy">画好UML图吧！不要每次费劲的想走到哪一步了，下一步是什么</li>
  <li data-pid="lzJijjfT">数据库结构关系图同上</li>
  <li data-pid="L9c3uvK_">Deadline的重要性不言而喻</li>
  <li data-pid="dYEEgqDa">细分任务到最小层级，对每个层级进行耗时预判，做好时间点规划（Mac下推荐OmniPlan）</li>
  <li data-pid="ZSTlk0nz">对项目中突然增加的可有可无的需求说NO</li>
 </ul>
 <p data-pid="n9KKcTkm"><b>测试驱动开发</b></p>
 <ul>
  <li data-pid="u7H9rLzH">单元测试是提升代码质量的最好方法。不要只是看看，赶紧去用起来</li>
  <li data-pid="27uS4jeo">代码覆盖率测试也是非常重要的，帮你找回潜在的隐患</li>
  <li data-pid="SoGFgtmQ">代码错误检查（代码右边的红红绿绿的不消灭掉以后就等着被坑吧）</li>
 </ul>

 <p data-pid="ZDhwNl6C">我们平常可能经常能看见各类人体工程学XX，比如人体工程系鼠标、人体工程学座椅！</p>
 <p data-pid="IVgxlX41">不可否认，大于5k一把的桌椅的确很舒服，但是像我这种屌丝买不起，怎么办？</p>
 <p data-pid="eBoaQgJn">对比1000内和3000以上的后我发现一些明显的<b>区别</b>（如上图）：</p>
 <p data-pid="jVzHHM0B">1.高端功能（比如座椅记忆功能，看着就高大上！但对于我这种预算有限的又没有实际需求还是pass）</p>
 <p data-pid="dDfFIOyn">2.靠手可以升降（公司的靠手最开始的时候经常会去调调，结果手也经常被卡着椅子和桌子间，后来发现靠手最合适的高度是手垂直放下，靠手与手关节交汇）</p>
 <p data-pid="VwVFjz8I">3.材质（透气度、安全度、是否异味）</p>
 <p data-pid="ogYj6CQZ">其实这个还挺在意的，比如透气度、安全度，但是当我详细的看过她的配置后我发现每一把椅子的安全是合格的，在每一把椅子坐垫下都有一块一样的钢板！透气度感觉公司的座椅坐久了屁股也会热，所以坐垫的透气度可以降级！</p>
 <p data-pid="GnUhTAfu">4.还有其它的一些小区别</p>
 <p data-pid="KSiM4wip">同时也发现一些<b>相似处</b>：</p>
 <p data-pid="u0oxT0Pb">1.座椅靠背的弧度一样（这是舒适的前提，必须保证！对比厚度也一样）</p>
 <p data-pid="DmauVdrO">2.都可以调节成躺着的姿势（连续工作一段时间可以调整椅子平躺着小憩一会）</p>
 <p data-pid="A3VRyvYp">3.底座都是铝合金的（建议不要选择钢质脚）</p>
 <p data-pid="1Redm_RE">根据我们的需求，发现透气度是最明显而又重要的区别，夏天不透气，真的不好受的，1000以下的透气效果不好怎么办？于是我就想到了退而求其次，买个凉席垫，使用后发现比其它透气效果都好！</p>

 <p data-pid="huvUibzm">其实里面功能大部分都能用世面上成熟产品替代的，但是那些工具无法根据我自己的意愿优化更改，所以我最终还是写了这么一个东西！</p>
 <p data-pid="Y3eeZpEN">主要常用的：</p>
 <p data-pid="LVbBhPAH">时间目标管理：类似TODO，但是根据我个人制定了一些月度、季度以及年度目标，然后从平常的待办事件中分析大目标的进展</p>
 <p data-pid="nN_LG4A0">日记本：这个后来还是废弃了，转投印象笔记了</p>
 <p data-pid="8FoZHg8R">我的钱包：后台也是懒得记了，闲麻烦，本来想做成读取账单的，后台发现账单太多而且又不全，最终还是放弃了</p>
 <p data-pid="732WXAm9">密码保险箱：这个使用频率还是非常高的，存了自己一百多个网站的账号密码等信息，密码采用随机高强度规则生成的，所以我大部分网站密码都是大于18位，且定期更改</p>
 <p data-pid="uJ6GCyoJ">人际关系：这个主要管理自己的人际关系，和待办事件里关联的，如果待办事件里@了某人，那么在人际关系中某人的事件里面就会出现！还有人际关系拓扑图以及一些生日提醒之类的</p>
 <p data-pid="KDdMyOqc">关注新闻：已废弃，转投Reeder。本来是想写成自己的RSS阅读器，结果发现做出来的没有什么优势</p>
