# 你们刚开始看英文文献是怎么看的？
- 点赞数：7888
- 更新时间：2020年01月14日09时28分57秒
- 回答url：https://www.zhihu.com/question/354245349/answer/973353833
<body>
 <p data-pid="f4INe-HX">研一的时候，帮师姐做实验，她拷给我一个文件夹，里面少说有两百多篇文献，而且大多是英文的，简直要疯了。但是硬啃了一段时间后，课题相关的文献一天能过五六篇，我就按照这几个阶段把读文献的过程梳理一下，把方法分享给大家。</p>
 <p data-pid="t9tPpbO5"><b>【第一阶段】</b></p>
 <p data-pid="L3qOM_kk">① 先<b>看相关的中文学位论文</b>(博士或硕士论文，本科就算了)~是学位论文不是期刊哦~这一步的目的是——<b>【扫盲】</b>。</p>
 <ul>
  <li data-pid="zVW01qed">因为学位论文，尤其是文献综述部分(都懂得，写论文最头疼的就是这块儿，七八千字码到吐)，归纳了我们需要掌握的知识点，解释了其中涉及的专有名词及意义，这很重要。</li>
  <li data-pid="SVEwhQ1K">之所以我们看不懂英文文献，很大一部分是对专有名词的陌生，即使用了有道词典翻译过来，也不知道讲的是啥。</li>
  <li data-pid="2hKkQFl8">另一方面，我们经常是好像看懂了英文，感觉心里明白，但用中国话翻译过来就驴唇不对马嘴，这就需要看看别人的论文是怎么翻译表述的。</li>
  <li data-pid="eCD9geX5">中文期刊因为受版面限制，重点放在创新方法和结果，介绍部分太简略，起不到扫盲的作用。</li>
  <li data-pid="dqgUZw2U">借鉴的论文不用太多，都大同小异，三五篇即可。不过每篇都有四五六十页，必须要咬咬牙啃下来，之后的阅读效率你会发现有质的飞跃。</li>
 </ul>
 <p data-pid="uzfmVst8">② sorry，我们还在第一阶段，因为还有一步更痛苦的，就是<b>通读英文文献</b>。这一步的目的是——<b>【扫英文盲】</b>。</p>
 <ul>
  <li data-pid="_60wTz6u">同样，三五篇即可，最好是文献综述，从头读到尾，遇到看不懂的就谷歌有道百度，然后和之前看的中文论文对照，从中找到准确的表达(说白了，就是说人话，别直翻)。</li>
  <li data-pid="Ceq2bZTg">然后别闲着，用笔在文献上标注出来，尤其是关键段落的翻译，这一步既方便日后回看的时候快速抓到要点，也能够很大地提高写论文的效率。</li>
 </ul>
 <p data-pid="GINg2Q9C">③ 通读还有一个目的——<b>【找套路】</b>。</p>
 <ul>
  <li data-pid="mbzrMfKw">中英文也读了快十篇，发现作者们反反复复也就围绕几个内容唠，无非表达上略有不同(查重，你懂的)。</li>
  <li data-pid="ZPTfG5Wb">归纳出这些点，找到套路，可能要花两三个月的时间，但这是值得的，读懂弄通，在进入下一阶段的时候，就会发现读的速度快的嗖嗖的。</li>
 </ul>
 <p data-pid="YEKbYhwM"><b>【第二阶段】</b></p>
 <p data-pid="lYgu8yUV">速读的时候就要<b>【带着目标定位】</b>，其实这一阶段主要还是一个<b>【筛选的过程】</b>。</p>
 <ul>
  <li data-pid="CYuKlmSI">拿来文献，先看题目和摘要，一两百个词，因为有了之前的知识积累，读起来so easy。然后扫一下正文的各个标题，直接跳到结论部分，同样一两百词。这就够了，文章的意义，研究的内容，和得到的结果都很清楚。接下来就要快速判断这篇文章：</li>
 </ul>
 <ol>
  <li data-pid="NtsV2Ckl"><b>和我的【论文框架】（目录）是否有关</b>，有关的留下来，无关的放在一边，完成【第一次筛选】——<b>定位文章</b></li>
  <li data-pid="zBZaC1-2">文章中哪部分<b>能帮我解决现在的困扰、提出一个新的思路、或者让我的文章更严谨更完整</b>，即进行【第二次筛选】——<b>定位章节</b></li>
  <li data-pid="6ZvQY7s7">通过速读+定位（主题阅读），我们能够大大提高浏览效率</li>
 </ol>
 <p data-pid="kmC_tvHv"><b>【第三阶段】</b></p>
 <p data-pid="Bj5Ufq8Z">确定了重点阅读的内容，下面就是要精读了，没有什么好方法，我当时就用了最笨的办法——<b>好记性不如烂笔头</b>。</p>
 <ul>
  <li data-pid="iNTgR3e3">把文献打印出来，一字一字读，用红笔划出重点句子，然后随手把翻译写在旁边的空白处。</li>
  <li data-pid="-oZ016UC">这个阶段我是不建议用电脑，因为读的过程中要思考和批注，用电脑感觉倒不方便还容易分散注意力，而且眼睛一直盯着显示器真的要瞎啦。</li>
  <li data-pid="zT_Uoe6L">读的文献多了以后，就会其中十分典型的几篇：要么是很详细的综述，能够全面且地介绍我研究的领域；要么是适合的框架，对文章的结构设计有很好的借鉴；要么就是好的方向，指明创新点；总之细数这么多文献，紧紧吃透这几篇就能够了，万变不离其宗。</li>
 </ul>
 <p data-pid="xAUKMxPv"><b>【第四阶段】</b></p>
 <p data-pid="SG62TMpV">花了大量笔墨在讲第一阶段，是真心觉得，花些时间做好知识储备很重要，而后面的阶段并没有固定的套路，只要对自己的研究有用就行。之所以分速读、主题阅读、精读，是帮助我们刻意练习，培养一种自然地筛选、吸收知识的方法罢了。</p>
