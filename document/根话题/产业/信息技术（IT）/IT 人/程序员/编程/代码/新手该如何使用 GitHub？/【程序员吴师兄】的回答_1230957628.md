 <h2><b>一、如何参与项目</b></h2>
 <p data-pid="8LEZdgZ0">如果你想参与某个开源项目，你首先要做的是先了解这个项目，最好的方式是先仔细阅读它的 README。</p>
 <p data-pid="0QX5SH_O"><b>一个标准的 README 应该包含以下内容：</b></p>
 <ul>
  <li data-pid="hW2z1QYP">项目背景</li>
  <li data-pid="YEcJreor">安装</li>
  <li data-pid="0Jq2Wn3W">使用</li>
  <li data-pid="ktuOObbA">Badge</li>
  <li data-pid="mhpmEktO">相关项目（可选）</li>
  <li data-pid="ZJk8CHdo">主要项目负责人</li>
  <li data-pid="6tA9dfjh">参与贡献方式</li>
  <li data-pid="INOEjX10">开源协议</li>
 </ul>
 <p data-pid="TUeGMGcj">通过<b>项目背景</b>可以了解这个项目动机和背景是什么，是为了优化什么问题还是因为市面上其它的项目太渣渣。</p>
 <p data-pid="nxE_vjNo">然后通过 <b>安装</b> 和 <b>使用</b> 知道项目如何运行起来，方便查看细节。</p>
 <p data-pid="UHhDpYZ4">最后就需要查看它的 <b>参与贡献方式 ，有的是</b>希望你提交 issue，有的是直接 push，有的是加群一起协作。</p>
 <p data-pid="Y7IknTm3">题主还问到 「核心的一些架构我暂时是力不从心的，那我还能为这个项目做什么呢」，这一点你不用担心，哪怕你只是修改里面的一个错误单词，一句错误的语法对于整个项目来说也是很有作用的。</p>
 <p data-pid="G_-WjQfj"><b>这不是开玩笑。</b></p>
 <p data-pid="m3EHCmZN">一个 4 岁小女孩就给 Linux 内核贡献提交。</p>

 <h2>为什么要修改这个字符？</h2>
 <p class="ztext-empty-paragraph"><br></p>
 <figure data-size="normal">
  <img src="https://picx.zhimg.com/50/v2-db029cf86debd93788dae42fb58fcd4c_720w.jpg?source=1940ef5c" data-caption="" data-size="normal" data-rawwidth="895" data-rawheight="643" data-original-token="v2-b95d2c5a2f53970b981cdd16b9ec8bda" data-default-watermark-src="https://pic1.zhimg.com/50/v2-0ce90285321c53d52a65bb88d695dabd_720w.jpg?source=1940ef5c" class="origin_image zh-lightbox-thumb" width="895" data-original="https://picx.zhimg.com/v2-db029cf86debd93788dae42fb58fcd4c_r.jpg?source=1940ef5c">
 </figure>
 <p class="ztext-empty-paragraph"><br></p>
 <p data-pid="sfGW-FyG">我们看到，正常情况下「--------」和上面的字符串是对齐的，但是刚好修改的那行是没有对齐的，所以是有问题的，是不严谨的。</p>
 <p data-pid="o-C8izHf">虽然是一个很小的修改，但是这个修改的意思非常大。</p>
 <p data-pid="HYbDsvOa">1、Linux 内核可以让所有人参与进来。</p>
 <p data-pid="8jAP8bxL">2、Linux 内核非常严谨，只要有错误，任何人都可以修改它。</p>

 <p data-pid="jZzmmBBs">非常细心的一个举动，这个修改在 riddit 上，很多人在激烈的讨论，有人认为这个修改没有什么作用，不就是一个字符而已，但是很多人对这个小女孩产生钦佩，觉得他是一个非常认真仔细的人。</p>

 <h2><b>二、编程初学者如何在 GitHub 寻找适合自己的小项目</b></h2>
 <p data-pid="8TPtRFFt">GitHub 成立于 2008 年，截止到 2015 年，GitHub 已经有超过两千八百万注册用户和5700万代码库。</p>
 <p data-pid="DmB5lApu">对于新手而言，要从这浩瀚的代码库中找到适合自己小项目无异于大海捞针，如果有个类似于 hao123 那样的导航网站就好了！</p>
 <p data-pid="mha0MLbR">登登登，还真有，一个国人建立的开源项目：<b>HelloGitHub，致力于分享 GitHub 上有趣，入门级的开源项目，对于编程新手而言十分友好。</b></p>
 <p data-pid="oRjf7Acj"><b>地址：</b><a href="https://link.zhihu.com/?target=https%3A//github.com/521xueweihan/HelloGitHub" class=" external" target="_blank" rel="nofollow noreferrer"><span class="invisible">https://</span><span class="visible">github.com/521xueweihan</span><span class="invisible">/HelloGitHub</span><span class="ellipsis"></span></a></p>

 <p data-pid="EOs5JcmM">在项目中，内容每月 28 号以月刊的形式更新发布，主要是面向编程新手、热爱编程、对开源社区感兴趣人群的项目。更新的内容主要包括：各种语言的流行项目、入门级项目、让生活变得更美好的工具、书籍、学习心得笔记、企业级项目等。</p>
 <p data-pid="qelUm1EU">这些开源项目大多都是非常容易上手、很 Cool，能够让你用很短时间感受到编程的魅力和便捷。从而让大家感受到编程的乐趣，动手开始编程。</p>

 <p data-pid="I1EGeUDJ">这些项目都有一些共同的特征，那就是很酷，非常容易上手的项目，编程的魅力和便捷体验起来就是这么简单。</p>
 <p data-pid="iZ3EOe2Z">对于编程新手而言，不管你是学 Java，还是学 Python，哪怕你是编程小白，不懂代码，也总能在里面找到适合你的项目。</p>
 <p data-pid="M6Htmsrf">这个新手项目还不够？那在推荐一个类似但更有趣的开源项目：GitHubDaily。</p>

 <p data-pid="RjoIjE8V">它分门别类的整理了好多有趣的项目，适合新手在零碎时间去阅读学习，了解 GitHub 的生态。</p>
