# 如何评价领导要用代码行数衡量每个人的工作量？
- 点赞数：5945
- 更新时间：2021年05月17日19时35分04秒
- 回答url：https://www.zhihu.com/question/295181406/answer/511572271
<body>
 <p data-pid="U1EEpslo">王二是一个非常优秀的程序员，别人100行代码才能完成的事儿，他往往10行搞定，别人一星期才能搞定的任务，他往往半天做完。王二不想揽那么多事情，就总是在每周快结束的时候才工作，其他时间都想办法娱乐。</p>
 <p data-pid="cl6zS_JN">领导很是看王二不惯，可又没什么办法。后来领导一统计，这王二的代码量是整个团队最少的。于是，有一天，领导就颁布了一条规定：<b>要按代码行评估绩效，每周统计大家的代码行数，平均前三加绩效，年中和年底根据绩效发奖金</b>。</p>
 <p data-pid="n8Q3CCLf">王二一听，心里明白怎么回事儿。不过他也不生气，很快想到办法：把一行代码能完成的功能，写成 10 行。比如一个给定两点计算矩形面积的函数，原本他写成下面这样一行代码：</p>
 <div class="highlight">
  <pre><code class="language-text">double getRectAreaByPoints(double x1, double y1, double x2, double y2){ return abs((x1 - x2)*( y1- y2));}</code></pre>
 </div>
 <p data-pid="ni0jAg_e">新规定颁布后，他会写成这样：</p>
 <div class="highlight">
  <pre><code class="language-text">double
getRectAreaByPoints
(
    double x1,
    double y1,
    double x2,
    double y2
)
{
    double width;

    width = x1 - x2;

    if(width &lt; 0)
    {
        width = -width;
    }

    double height;

    height = x2 - y2;

    if(height &lt; 0)
    {
        height = -height;
    }

    double area;

    area = width * height;

    if(area == 0)
    {
        return 0;
    }
    else
    {
        return area;
    }
}</code></pre>
 </div>
 <p data-pid="zSCLD9q-">一行变40行！</p>
 <p data-pid="EsFPlHkh">王二心想，娘希匹，看哪个龟孙的代码行数能超过老子！</p>
 <p data-pid="sGACv_g0">果然，年中一算绩效，我的天呐，王二拿的奖金最高！</p>
 <p data-pid="Zy55Pn1j">领导一看，不对呀，以往王二的代码量最少。于是，他就来查代码。这一查，勃然大怒，看出其中蹊跷，就取消了王二的奖金，还罚了王二几百块钱。</p>
 <p data-pid="HyiS942p">领导出了口气，不想和王二计较了，可又不想随便废掉新政，觉得那样太下不来台，就左思右想，琢磨改进的事儿。</p>
 <p data-pid="aEm7brfk">终于，给他想到一个绝妙的办法，于是颁布了一个新规定：<b>还是按代码行数评估绩效，规则更改为，在完成工作任务的同时，谁的代码行数最少，谁的绩效最高；谁的代码行数最多，谁的绩效最低。</b></p>
 <p data-pid="jfGnxjbE">这下，很多程序员傻眼了，没办法，还得适应新龟腚呀，就开始改变，每天都猛琢磨，怎么把原本200行代码可以实现的功能用一行来完成。</p>
 <p data-pid="ztL8QASz">结果呢，整个团队，全变成了思考者，很多人一个月也不写一行代码，一年写不了10行。王二更绝，一年就写一行！</p>
 <p data-pid="NBQzetqZ">最后，大家任务都没完成，谁也没拿到奖金。</p>
 <p data-pid="NbkUF_4v">这下，领导又发飙了！</p>
 <p data-pid="PeW2om7g">于是，苦思冥想一昼夜，颁布了一个新规定：<b>还是按代码行数评估绩效，规则更改为，在完成任务的前提下，代码行数量越靠近均位数，拿的奖金越多。</b></p>
 <p data-pid="HPG0lt0N">这个法令一出，大家死活想不到办法，整日里愁眉苦脸。</p>
 <p data-pid="gXD70gIi">要说还是王二聪明，熬了一夜，抽掉13包长沙后，写出来一个代码行归一工具：只要你输入一个数字，这个工具就能把你的代码拆成那么多行。比如你输入300，它就可以把你的一行代码拆成300行，还不影响功能。</p>
 <p data-pid="FhoqrsWw">王二乐于助人，编译了一个写死300行的工具发给大家使用。这样，每个人提交代码前，都用这个工具跑一下，工具自动完成折行、加空行等操作，把代码行数调整到300行。这样，大家每周代码行数量都变成了300行，半年下来，都有望拿到最高奖金。</p>
 <p data-pid="8GBMRpE-">然而，领导狂飙啦！天底下就不该有这种事啊！</p>
 <p data-pid="SXzrqpOH">于是，领导就又来查代码，结果发现，每个人的代码都跟加过扰似，根本读不懂！</p>
 <p data-pid="g-fZlJji">最后，领导两天两夜没睡觉，第三天早上，发布了新规定：废除按代码行数评估绩效。</p>
 <p data-pid="hKS4TQIa">大家一听，都心安了。</p>
 <p data-pid="XX96qml3">然后好景不长，第四天，领导颁布了一条新规定：<b>从今天开始，大家的绩效，按每千行代码bug数来计算，bug越多，绩效越低，bug越少，绩效越高。</b></p>
 <p data-pid="h2T4rtJM">王二一听，哇呀，这下难了，可他心高气傲，心想老子还能想不出办法来？于是每日里别事不干，就琢磨怎么降低每千行代码bug数……</p>
 <p data-pid="EXWIYzvk">一晃半年过去，王二一行代码没写，结果却发生了意料不到的事情：他的每千行代码bug数为0，最低，绩效最高！</p>
 <p data-pid="Fh2GzPPa">王二大笑三声，跑去找领导要奖金，领导难以承受巨大的冲击，狂喷三分钟鲜血，被120送走了……</p>
