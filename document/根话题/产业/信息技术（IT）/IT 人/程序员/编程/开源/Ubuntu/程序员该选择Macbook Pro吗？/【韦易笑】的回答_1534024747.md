# 程序员该选择Macbook Pro吗？
- 点赞数：10282
- 更新时间：2020年10月23日01时50分40秒
- 回答url：https://www.zhihu.com/question/362640359/answer/1534024747
<body>
 <p data-pid="OhMj674c">知乎上清一色的给 mac 唱赞歌，都 2020 年了，就没人出来说一句实话么？</p>
 <p data-pid="uCVNNwwG">某答案里居然连“一个程序员不买mac就是不思进取”这种言论都有人说得出来？我说，现在都 2020 年了，<b>除非你非要用 xcode，我找不到程序员有啥必须用 mac 的理由</b>。</p>
 <p data-pid="ZqIFiLU4">我用过两台 MacMini（家用，公司用），还有一台 Macbook Pro，最早开始用 mac 也是十多年前的事情了，折腾了 3 台设备后，现在坚定的用回 Windows，理由有几条：</p>
 <p data-pid="GD8Pbpon">1. 强制商店的 curated 模式，适合普通内容消费者但是<b>不适合作为生产力工具</b>。</p>
 <p data-pid="R-_ATVGk">2. 越来越<b>朝着大号 iPad 模式发展</b>，以后想体验，还不如直接买 iPad + 键盘。</p>
 <p data-pid="1xtF7V0o">3. 硬件锁死，<b>无法自由搭配硬件，</b>想换 cpu ？没门，想用好显卡？没门！一切都要听我苹果的，不想跟着我转 arm ？不可能的事情。</p>
 <p data-pid="p-bttqRd">4. 系统落后，比起 windows 桌面而言。</p>
 <p data-pid="H17Q9vpw">5. 唯一优势就是字体好看点，这点<b>在高分屏下，字体根本不是事情</b>。</p>
 <p data-pid="rbDaMQJH">6. 同各种外设搭配不好，很多外设无 mac 驱动，无法用 n 卡 egpu。</p>
 <p data-pid="ll_ady39">7. CPU 发热厉害，随便开个网页，风扇就吹不停，刷知乎都会把本子刷烫。mbp 那个风扇，每次一响，前后左右的人都回头看我，以为我上班时间带薪吹头发呢。</p>
 <p data-pid="0LAQP7-f">8. 品控问题，系统 bug 越来越多。</p>
 <p data-pid="v2VPRETl">9. 软件厂商正在远离，不少开发商已经很头疼 mac 了，每次系统升级都需要为 mac 花费一个月以上做软件移植，各种操作系统 bug 扯皮扯半天。 同样软件 mac 下更新最慢，不少一年多无更新。</p>
 <p data-pid="0pFNeaO7">10. 兼容性，windows 10 还能跑 adobe photoshop 6.0 ，而最新的 mac 系统，甚至连很多两年前刚刚发布的软件都跑不起来。</p>
 <p data-pid="E6JhDaL3">11. 触控板方便？这个真的不是什么刚需，Win10 触控板比 Win7 已经好了非常多，两指，三指操作丝滑，mac 的触控板已经没有那么大的优势了。</p>
 <p data-pid="ulF9C69p">12. 垃圾软件少？这是过去时了，Win10 商店里虽然良莠不齐，但起码的审核还是有的，不少好用的工具还是找得到的，Win10 有商店给你用，但是不强制你商店，这是我比较喜欢的。再，作为程序员，基本的防止垃圾软件的能力还是有的吧？不至于像小白那种动不动就弄个全家桶吧？实在拿不准，Win10 还有 sandbox，你可以先再 sandbox 里面运行下看看。</p>
 <p data-pid="O2oCpovK">（补充：PC本硬件自由不是说你可以撬 CPU，而是有无数品牌给你选，每个品牌又有无数型号，你有权利随便选择心仪的硬件，mac 呢？对不起，你只有一条路，听苹果爸爸的，想换你就只能撬了，比如，你换个 amd 的 cpu 给我看看？）</p>
 <p data-pid="iary_XkL">--</p>
 <p data-pid="QjzttT9P">对于服务端开发人员适合 Mac 的言论，这是老黄历了，那套基于 bsd 的 darwin 和线上 linux 服务器差距之大，用过的都知道，人家 windows+wsl 都能和服务端做到二进制兼容了，你 darwin 连若干基础命令的参数你都没法和 Linux 服务端保持一致，很多脚本都要专门为你做一遍移植，连脚本层的兼容都做不到，何德何能去跟二进制级别兼容的 wsl 比呢？</p>
 <p data-pid="6i713n6A">给你们举个例子，<code>realpath</code> 命令，返回文件名的真实全路径，Linux 是有一个 -s 参数的，可以不解析软连接，很多情况下就比较方便，特别是同一个工程目录下面的文件，可能用软链接引用了外部的文件，我希望得到它在工程目录内的绝对路径，而不是外部的路径，macos 下面就比较惨了，不支持这个参数，需要调用 python 的 abspath 来做类似的事情，这开销就比较大了。</p>
 <p data-pid="GkjT_0az">稍微用下 mac 你都会发现里面的基础命令跟 Linux 完全没法比，不是版本太老，就是功能不够全，很多工具你加个 <code>--help</code> 看看，你会发现 mac 下面的同名工具，功能上差 Linux 不止一个数量级，何况很多残缺。 而这些，直接用 Linux 或者用 WSL 都完全没有问题。</p>
 <p data-pid="UKCb1W3t">你服务端开发者用 windows/ubuntu 我都觉得还行，用 mac ？我想不出来任何优势和理由。</p>
 <p data-pid="DG78zaXL">客户端开发如果不是 xcode，更没有任何理由用 mac。</p>
 <p data-pid="oxBXjs2_">说终端漂亮的，iterm 2 觉得很好看是吧？延迟大的一塌糊涂，你和 <a href="https://link.zhihu.com/?target=http%3A//terminal.app/" class=" wrap external" target="_blank" rel="nofollow noreferrer">terminal.app</a> 对比下你都会发现打字有明显的延迟，难以忍受按下键盘还有几十到 100 毫秒的时间字符才显示得出来。 Windows 10 的 cmd.exe 窗口现在已经是完全兼容 xterm 终端了，而且还有更快更漂亮的 ms terminal，iterm2 在 ms terminal 面前，真的是幼儿园的水平，ms terminal 速度比 iterm2 快 5 倍以上。</p>
 <p data-pid="VHfEP_gp">不对比就没有伤害，延迟比较："ms terminal &lt;&lt;&lt;&lt; <a href="https://link.zhihu.com/?target=http%3A//terminal.app/" class=" wrap external" target="_blank" rel="nofollow noreferrer">terminal.app</a> &lt; iterm2" ，我用多了 iterm2 没发现，偶然用回 <a href="https://link.zhihu.com/?target=http%3A//terminal.app/" class=" wrap external" target="_blank" rel="nofollow noreferrer">terminal.app</a> 发现，我靠，真快。切换到 ms terminal 以后就发现，原来还可以快到这种程度，于是再也回不去了。很多觉得 iterm2 够用了，不需要更快的，是因为他们没有用过真的好东西，就像<b>以前我觉得 1024x768 的显示器已经非常高清了，完全够我用了，但是当我看了 2k 的显示屏以后，我就再也回不去了一样</b>。</p>
 <p data-pid="v8guJ-Wy">价格一万多的 macbook pro，盘给你配 256GB，还没我只装操作系统的 C 盘大，我现在 windows 本子随便配置了下，1.5T 我都觉得不够用，好在我的本子可以插 2 条 M.2 和 1 块 2.5寸的 HDD，不够随便加。现在如果不是强制用 xcode ，买 macbook 真的花冤枉钱买破烂货，纯粹给信仰充值用的。</p>

 <p data-pid="sAm-Iq8z">你就连同一款软件很多也是 mac 版比 windows 版本贵，不但贵，而且更新慢，bug 多，修复不及时，用了干嘛？别的同事发给我的 ppt，word 文档，用mac的办公三件套打开经常显示不正确，被迫买套 mac 的 office。</p>
 <p data-pid="NjG-3kU_">今天强迫你升级系统，不然软件用不了；明天把你 ESC 实体键去掉；后天告诉你，硬件已用不了最新系统，该买新硬件了；大后天跟你讲 256GB 其实够你用了；再大后天问都不问你就把你绑架到 ARM 那里去了，我想问一句那些天天被苹果扇耳光，还大呼苹果爸爸好的开发者们，被虐久了会有快感了吗？哪家 PC 本子敢这么搞，早被骂死了，mac 用户的耐受力果然很强！！</p>
 <p data-pid="tIknetWZ">所以，除非你被苹果绑架，必须用 xcode，否则真的想不出任何用 mac 的理由来。</p>
 <p data-pid="peZbv7w6">还是那句老话，随着 mbp 的 ipad 化进程，以后真想用，买个键盘接在 ipad 上就可以体验了。</p>

 <p data-pid="N51TDv2M">部分评论解答：</p>
 <p data-pid="y56kbCfO">第一，评论区有些人津津乐道，哪家公司给员工配 mac 了，哪家公司员工清一色 mbp 啦，这能说明什么？员工不少以前都是用 PC 的没有用过 mac 好奇想体验下很正常，公司也乘机发两篇 PR 稿子吸引下不明真相的潜在求职者，这其实是好事，等他们用两年就会发现问题了，sooner or later。</p>
 <p data-pid="fdEwr9oJ">第二，评论区一些人说自己工作就是连着服务器，在服务器上搞，配置高低不在乎，首先，人家问的是程序员的开发机，没问你这种上网本，开发机就是要能本地开发的东西，是要能开发东西的，不是离开了 server 就什么都做不了的上网本；其次，就算你天天连着公司服务器写代码，那么当你要做点自己的项目呢？当你想刷两题 leetcode 呢？莫非你还想连到公司服务器上干私活不成？最后，只能说明你工作内容太单一了。</p>
