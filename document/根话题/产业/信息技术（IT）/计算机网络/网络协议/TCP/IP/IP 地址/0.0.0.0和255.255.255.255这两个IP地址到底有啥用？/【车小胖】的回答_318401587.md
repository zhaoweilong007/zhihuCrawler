# 0.0.0.0和255.255.255.255这两个IP地址到底有啥用？
- 点赞数：7927
- 更新时间：2021年11月14日11时36分36秒
- 回答url：https://www.zhihu.com/question/267097519/answer/318401587
<body>
 <p data-pid="Prtlb6fR"><b>IP地址</b><br>
  *<b> 0.0.0.0</b><br>
  当主机启动时，如初生的婴儿（<b>0.0.0.0</b>），没有身份信息，需要向外界大声哭泣，“大神，请赐予我一个名字吧”！（<b>Discovery</b>)</p>
 <p data-pid="hxGHEzJ4">婴儿的呼唤被广播了出去，使用的是255.255.255.255这个地址。</p>
 <p data-pid="n_zFm65i">* <b>255.255.255.255</b><br>
  255.255.255.255是一个特殊的广播地址，意味着所有的主机，有同学会好奇地问，是否是指互联网上所有主机？</p>
 <p data-pid="3L6m6KoH">No！因为255.255.255.255传播范围非常有限，到了广播域的边界（<b>网关</b>）会自动终结，如果把广播比作谣言，那么网关是什么呢？</p>
 <p data-pid="H8GAQTzE">“谣言止于智者”中的<b>智者！</b></p>
 <p data-pid="wPxlKQjG"><b>那么能听到婴儿哭泣的都有哪些人？</b></p>
 <p data-pid="j0zhksO7">* <b>婴儿自己</b><br>
  婴儿听到也没有用，因为无法给自己起名字，左耳进右耳出。</p>
 <p data-pid="IWxl6Rr0">* <b>大神</b><br>
  大神，就是这里的DHCP Server，专业给别人起名字，应答了婴儿的啼哭，是这么回应的：<br>
  “小宝贝，你的名字是192.168.0.2，喜欢吗？”（<b>Offer</b>）</p>
 <p data-pid="PBV5h5QP">* <b>广播域其它主机</b><br>
  其它主机也听到了，但是由于没有起名字（DHCP Server / Listening UDP Port 67）的功能，当作什么事没有。</p>
 <p data-pid="Wk29oH__"><b>婴儿的反应</b><br>
  婴儿听到了大神的天籁之音，停止了哭泣，于是很快乐地回复了大神的美意（Offer）</p>
 <p data-pid="Umdvppld">“大神，我很喜欢192.168.0.2这个名字，就它了” (<b>Request</b>)</p>
 <p data-pid="O_STrUsm"><b>大神最后的确认</b><br>
  看到了小婴儿接受了自己的推荐的名字，大神很愉快滴回复了小婴儿：</p>
 <p data-pid="Wgu-AB_L">“祝一切都好，再见！” （<b>ACK</b>）</p>
 <p data-pid="A9RXZpnJ">大神为了防止 192.168.0.2这个名字被其他小朋友使用，将这个名字标记为“<b>已使用</b>”状态。</p>
