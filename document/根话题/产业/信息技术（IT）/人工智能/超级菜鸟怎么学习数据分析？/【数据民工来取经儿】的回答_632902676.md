 <p data-pid="puPW-lxG"><b>数据分析入门, 可以很简单。</b></p>
 <p data-pid="l-PatoNq">最初级的分析师，Excel + 细分/对比的基本思维</p>
 <p data-pid="s8ITA5yL">这么简单, 岂不是太没竞争力了？<b>别慌！看完..再决定慌不慌</b></p>
 <p data-pid="pJICKgzS">第一, 数据处理, Excel只是最基础的能力， 当然也是必备的。必备是因为使用频率真的很高，很多同学可能觉得只有code才代表能力，千万别有这个不成熟的想法，比如飞机火车出现后，自行车依然是一种极大提升人类出行效率的交通工具。Excel的熟练程度，也会极大提升数据分析过程中的效率，大量的时间成本就是这样被节省的。</p>
 <p data-pid="sWGxTAcL">第二，Excel功能太多，出版的书都好几本新华字典的厚度，我们不能也完全没必要学习所有的功能，则需而取。下面几个关键词(排名不分先后)，你如果一看会心一笑那说明就够了，如果懵逼，就说明你还需要学习下它们: 透视表 ，刷选， 多列排序，折线图，直方图，双坐标图，数据导入分隔符设置， vlookup， count， sum， 冻结首行</p>
 <p data-pid="0oHJdyBN">第三，细分和对比的思维。数据进行分析，基本、必用的方法，是否能用好，跟自己数据敏感度和业务熟练度强相关。如果你觉得这个也很简单，我不认为你错，但是请把它用好。同样，下面我提几个问题，你看看是不是胸里竹子成林:</p>
 <ol>
  <li data-pid="wa3GIe5l">今天，煎饼卖出去50张， 评价卖的怎么样？</li>
  <li data-pid="G3g7-zu2">今天，知乎App活跃1000万， 评价是否正常？</li>
  <li data-pid="Dgk8k2K7">今天，煎饼出摊时间也是3小时，为啥收入少了100元， 为什么？</li>
  <li data-pid="A1LbnJFh">今天，知乎App活跃用户跟昨天一样多，知乎盐会员比昨天少买了100万， 为什么？</li>
 </ol>
 <p data-pid="VKQ6jP1Y">以上，问题，第一步，绝对是细分和对比。煎饼摊主，知乎数据分析师，会针对自己业务展开大扫除的分析。</p>
 <ul>
  <li data-pid="u0pzWCDh">我是煎饼摊老板:</li>
 </ul>
 <ol>
  <li data-pid="S8oIj0pm">数据敏感度: 今天跟昨天比下降比例？今天出摊时间多久，与往常降低多少小时？</li>
  <li data-pid="GnGOZhDu">业务熟悉度: 今天是周末吗？ 对面多了个卖肉夹馍的？</li>
 </ol>
 <ul>
  <li data-pid="7bPOG7-8">我是知乎数据分析师:</li>
 </ul>
 <ol>
  <li data-pid="0Id77xyK">数据敏感度: 下降了多少？昨天是周几？</li>
  <li data-pid="Loknxtr1">业务熟悉度: 昨天有活动吗？这段时间新增高质量live数量怎么样？</li>
 </ol>

 </ol>
 <h3>第三、常用分析方法</h3>
 <ol>
  <li data-pid="W8fm3iXs">对比: 大法宝, 关键要知道怎么对比和谁对比</li>
  <li data-pid="ba3JOl3z">细分: 分地点，分人群，分时间，分产品，分渠道</li>
  <li data-pid="HNFUiz8v">漏斗: 每一步的转化率, 清晰有层次</li>
  <li data-pid="AxRsUvSy">RFM: R最近消费日期, F消费频次, M消费金额. 这个可用来粗暴的进行用户分层，已经被广泛用在各个行业</li>
  <li data-pid="iVexpkz8">TGI: 衡量某类人群某个特征与整体人群该特征平均值进行比较, 量化该人群该特征是否强大或弱小</li>
  <li data-pid="nBCTfLku">AARRR: Acquisition用户获取,Activation活跃,Retention留存, Revenue收入, Refer传播</li>
 </ol>

 <blockquote data-pid="SgcVDzlT">
  常用统计指标
 </blockquote>
 <ul>
  <li data-pid="zkCtwcKT">平均值: 不解释</li>
  <li data-pid="O7zboFbs">中位数: 假如小明班级30人, 成绩排名15的成绩分数值就是小明班级成绩分数的中位数</li>
  <li data-pid="3smwgcka">p值: 一句话粗暴解释, 用于假设检验, 其值反映某一事件发生的可能性大小. 假设A成立，然后找出支持A不成立的概率总和, 如果该值小于0.05(一般是这个阈值, 当然可以自己定义), 说明小概率事件发生, 我们认为A不成立</li>
 </ul>

  场景化思维习惯
 </blockquote>
 <p data-pid="OhP95hE1">数据背后对应的是人与事的场景。用户在什么场景下购买？为什么用户不付费？有些可能数据上看不到，但我们可以按照认知推断。</p>
 <p data-pid="p5JiLGRZ">拿自己来说，上班从回龙观到西三期6公里，公交和地铁都不太方便，所以会选择骑车，这是使用场景。另外，自己比较注重时间成本，能提升自己效率缩短时间的服务，自己乐于付费，所以不会因为充值小黄车后，就不购买摩拜月卡，相反，连小蓝、哈罗的月卡也都会购买，因为不希望花费时间找车(都是趁运营MM搞活动的时候大肆购买，哈哈)。然而，老婆是典型的羊毛党，坚持贯彻将互联网服务免费到底，享受着烧钱大战期间的各种赠送，坚决不买会员，但是当优惠消失，自己又确实需要时，才会临时付费，这就是背后的场景。我和朋友代表的就是两大类用户群体，因为不同的价值观，自然对应不同的用户行为。</p>
 <p data-pid="3TqiRbnv">数据分析要学会分析和拆解问题的思维，这一点麦肯锡咨询资深顾问，巴巴拉·明托写的《金字塔原理》已经成为数分每位同学必读书籍。</p>
