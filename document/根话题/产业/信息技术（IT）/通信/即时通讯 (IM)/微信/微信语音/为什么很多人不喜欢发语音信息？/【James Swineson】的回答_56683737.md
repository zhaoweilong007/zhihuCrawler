# 为什么很多人不喜欢发语音信息？
- 点赞数：9369
- 更新时间：2015年07月29日00时04分55秒
- 回答url：https://www.zhihu.com/question/22851866/answer/56683737
<body>
 <p data-pid="4aYHfY65">先表明立场：我极其反感滥用语音。</p>
 <blockquote data-pid="IacJf8e6">
  浪费时间就是谋财害命。——鲁迅
 </blockquote>
 <p data-pid="2WxlkAZK">语音的信息量远大于文字。信息量大会带来许多不便，比如：</p>
 <ul>
  <li data-pid="zIgm3vix">人工分析和筛选需要更多的时间和精力。<br>
   我一眼扫过去能看完的文字（呃我小学低年级就能达到每分钟千字的阅读速度了），听语音的效率降低至少十几倍。何况花这十几倍精力听完的东西还可能一点意义都没有。更何况如果听不清（发语音的人并不知道最终效果如何），效率再降低几十倍。</li>
  <li data-pid="30C6vr37">听觉相对视觉更难更切换上下文（context）。<br>
   你看网页的时候有人给你 QQ 发了句话，你可以轻松看了回复了然后接着看网页（只要不是啥重要的事情）；但如果有人在你打电话的时候给你发了句语音（而你又不得不听），你肯定卡壳喽。</li>
  <li data-pid="cYkJ2FYt">音频流暂停的成本更高。看文字，我可以随时停下来；而音频的暂停和恢复至少需要一次操作的成本。对于某些不能拖进度条的应用（说你呢，微信），长语音一旦停止就得从头听一遍。这儿还有个社交上的优先级问题：比如你在跟女神共进晚餐，有人给你发了条短信，你可以瞟一眼判断事情的严重性，一般的事情可以暂时忽略；有人给你打个电话，你肯定得说不好意思我接个电话然后离席打吧；而若是有人给你发了条语音，你是边听女神讲话边听语音呢（结果两边都听了个一知半解），还是说不好意思我听个语音，然后走到一边听完呢？</li>
  <li data-pid="chWaTDvl">难以自动化分析和处理。保存和搜索文字十分简单，音频你试试看？别跟我说语音转文字，那准确度尚不堪用。以及，微信在语言非中文的环境下不提供语音转文字功能。<br>
   -“对方讲了啥？截个图我看看”<br>
   -“……”</li>

 </ul>
 <p data-pid="yM7VpB2J">还有一点：声音会不可控地扩散到物理空间中。下面的场景很常见：</p>
 <ul>
  <li data-pid="QGk9lajk">公共场合听语音，外放了，被一群人听到</li>
  <li data-pid="SdYxQyIQ">公共场合听语音，用听筒，还是被一群人听到</li>
  <li data-pid="7rvpGzWK">公共场合听语音，用听筒，发现听不清，于是不得不调音量再听一遍</li>
  <li data-pid="9BTE16Xp">公共场合听语音，用听筒，发现最大音量还是听不清，于是用外放，手机喇叭贴在耳朵上<br></li>
  <li data-pid="RKWDDn8w">安静环境听语音，用听筒，发现音量太大，耳朵聋了，于是不得不调音量再听一遍</li>
  <li data-pid="hmu5khY4">安静环境听语音，想用听筒，结果不慎用了喇叭，耳朵聋了，于是慌忙按暂停</li>
  <li data-pid="PM6QjrHl">听语音，想用听筒，结果没听到声音，一看，原来是脸碰到屏幕暂停了</li>
  <li data-pid="GgB9BZTO">听语音，想用听筒，结果没听到声音，一拿开，外放了，被一群人听到<br></li>
  <li data-pid="uKEpXRlr">开会、上课的时候时候收到语音，恨不得隔着屏幕掐死对方（然后偷偷摸出耳机带上并期待老板/老师没看到）</li>
  <li data-pid="DX-1Rkn8">收到语音，一时不能听，于是就忘了……忘了……</li>
  <li data-pid="3R48U8x1">……</li>
 </ul>
 <p data-pid="q4GAr2a-">所以我现在用语音就两个场合：</p>
 <ul>
  <li data-pid="7f7z_AUX">没法打字（比如在骑车或者开车）</li>
  <li data-pid="aAu3HKbw">没有更合适的途径（比如要唱首歌给别人听）</li>
 </ul>
 <p data-pid="QUZgFmRB">剩下的时间一律打字。你要听语音，自己手机上装个文本转语音（TTS）的软件不就行了。</p>
