 </figure>
 <p data-pid="NLMBYi99">首先看区域 1，这里的绿色三角形明显就是 <code>.vue</code> 文件……所以这就是个典型的 <a href="https://link.zhihu.com/?target=https%3A//vuejs.org/" class=" wrap external" target="_blank" rel="nofollow noreferrer">Vue</a> 前端项目嘛。再脑补一下路径名，一下什么 src / assets / components / services / views 名字都齐了，前端同学们肯定不会陌生。</p>
 <p data-pid="zvZ_sX6J">然后看区域 2 和 3，这看来无疑是 JavaScript 无误了。这部分代码里透露了什么信息呢？整理如下：</p>
 <ul>
  <li data-pid="Efmfn-vB">区域 2 里多次出现了疑似 <code>self</code> 的变量名，这属于多年前的老黄历写法，说明这项目大概率历史比较悠久。</li>
  <li data-pid="ewJNFvF8">区域 2 最上面封装的函数名字像是 <code>getCommunityData</code>，这类请求接口数据的 API 用的却是最裸的回调函数，说明代码质量堪忧。</li>
  <li data-pid="Z62FRMKO">区域 3 里大量使用 JS 里早该被废弃的 <code>==</code> 而不是 <code>===</code> 做比较，说明编码规范很不严谨。</li>
  <li data-pid="_90-k8Zd">由代码缩进可知，整个区域 2 和 3 外面至少还包了三层函数。这说明代码可能是硬堆起来的，常见于赶进度的项目里。</li>
  <li data-pid="g5NYdVkd">对比区域 2 和 3，短短半个屏幕里就出现了 <code>for-in</code> 和 <code>forEach</code> 两种不同的循环方式，说明项目很可能多人接手过。</li>
 </ul>
 <p data-pid="ZQzp-ptj">这么多线索暗示下来，小哥维护的这大概是个典型的屎山项目……能在一屏代码里传达出这么多要素，向小米宣传团队的专业态度致敬！</p>
 <p data-pid="dGMS3wLh">另外图中 <code>id: val.</code> 这一行有典型的「还没写完」式错误提示，表明小哥已经在实际编码中。而此时手机上的时间是 10:33，说明他在接到电话的三分钟内就进入了编程状态，<b>时刻都没有放松下工作</b>。</p>
