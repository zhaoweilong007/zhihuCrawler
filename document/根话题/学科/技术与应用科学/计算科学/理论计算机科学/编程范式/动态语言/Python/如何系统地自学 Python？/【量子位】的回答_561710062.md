 <p data-pid="fqIQ7kmV">每一个示例的结构都是一样的，以“Mutating the immutable!/强人所难”为例：</p>
 <p data-pid="GppuyKRz">首先，会给出代码：</p>
 <div class="highlight">
  <pre><code class="language-python"><span class="n">some_tuple</span> <span class="o">=</span> <span class="p">(</span><span class="s2">"A"</span><span class="p">,</span> <span class="s2">"tuple"</span><span class="p">,</span> <span class="s2">"with"</span><span class="p">,</span> <span class="s2">"values"</span><span class="p">)</span>
<span class="n">another_tuple</span> <span class="o">=</span> <span class="p">([</span><span class="mi">1</span><span class="p">,</span> <span class="mi">2</span><span class="p">],</span> <span class="p">[</span><span class="mi">3</span><span class="p">,</span> <span class="mi">4</span><span class="p">],</span> <span class="p">[</span><span class="mi">5</span><span class="p">,</span> <span class="mi">6</span><span class="p">])</span></code></pre>
 </div>
 <p data-pid="xOc_I-pH">然后，给出Output（ Python version）：</p>
 <div class="highlight">
  <pre><code class="language-text">&gt;&gt;&gt; some_tuple[2] = "change this"
TypeError: 'tuple' object does not support item assignment
&gt;&gt;&gt; another_tuple[2].append(1000) # 这里不出现错误
&gt;&gt;&gt; another_tuple
([1, 2], [3, 4], [5, 6, 1000])
&gt;&gt;&gt; another_tuple[2] += [99, 999]
TypeError: 'tuple' object does not support item assignment
&gt;&gt;&gt; another_tuple
([1, 2], [3, 4], [5, 6, 1000, 99, 999])</code></pre>
 </div>
 <p data-pid="fqPCAjcl">然后，对意外输出的结果进行简短的描述，在这个示例中，就是：</p>
 <blockquote data-pid="mi6_ssho">
  我还以为元组是不可变的呢…
 </blockquote>
 <p data-pid="p8orjKIB">接下来，就会对示例进行说明，简要叙述发生了什么以及为什么会发生。如有必要, 也会举例说明。</p>
 <p data-pid="B2Qrfnd9">在这个示例中是这样的：</p>
 <ul>
  <li data-pid="iNdud1Yd">引用<br><a href="https://link.zhihu.com/?target=https%3A//docs.python.org/2/reference/datamodel.html" class=" external" target="_blank" rel="nofollow noreferrer"><span class="invisible">https://</span><span class="visible">docs.python.org/2/refer</span><span class="invisible">ence/datamodel.html</span><span class="ellipsis"></span></a></li>
 </ul>
 <blockquote data-pid="9MVQ0_Sx">
  不可变序列 不可变序列的对象一旦创建就不能再改变。(如果对象包含对其他对象的引用，则这些其他对象可能是可变的并且可能会被修改; 但是，由不可变对象直接引用的对象集合不能更改。)
 </blockquote>
 <ul>
  <li data-pid="hYYi65gQ">+= 操作符在原地修改了列表. 元素赋值操作并不工作, 但是当异常抛出时, 元素已经在原地被修改了。</li>
 </ul>
 <p data-pid="NU-J8Sfe">有些地方，贡献者还会给出译注，比如整个示例中就是：</p>
 <blockquote data-pid="hwAh8LgL">
  对于不可变对象, 这里指tuple, +=并不是原子操作, 而是extend和=两个动作, 这里=操作虽然会抛出异常, 但 extend 操作已经修改成功了。
 </blockquote>
 <p data-pid="tm6X3pmz">其他还有50个示例，等你来看~</p>
 <h2><b>怎么使用？</b></h2>
 <p data-pid="ADZXmA0E">当然，要学习一下怎么使用这个资源。项目贡献者在用法部分表示，最好依次阅读下面的示例。</p>
 <p data-pid="SP3CuoZI">然后，在阅读每一个示例的时候，这样做：</p>
 <blockquote data-pid="FumazbrA">
  仔细阅读设置例子最开始的代码。
  <br>
  阅读输出结果。
  <br>
  确认结果是否如你所料。
  <br>
  确认你是否知道这背后的原理。
  <br>
  如果不知道, 深呼吸然后阅读说明 (如果你还是看不明白, 别沉默！可以提问题)。
  <br>
  如果知道, 给自己点奖励, 然后去看下一个示例。
 </blockquote>
 <p data-pid="bJqa-JzS">此外，还可以在命令行阅读 WTFpython，有 pypi 包 和 npm 包(支持代码高亮)，不过都是英文版的。</p>
