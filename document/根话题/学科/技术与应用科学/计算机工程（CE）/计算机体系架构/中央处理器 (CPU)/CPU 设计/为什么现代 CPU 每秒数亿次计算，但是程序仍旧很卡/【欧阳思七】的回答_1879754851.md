# 为什么现代 CPU 每秒数亿次计算，但是程序仍旧很卡?
- 点赞数：5633
- 更新时间：2022年07月10日17时47分06秒
- 回答url：https://www.zhihu.com/question/458730114/answer/1879754851
<body>
 <p data-pid="r1f-ngek">我这样拉跨程序员写的程序经常这样:</p>
 <ul>
  <li data-pid="_LUOjXan">不做异步,就在UI线程上搞事情.任何事做完之前.程序都是卡的。</li>
  <li data-pid="3HDUrLzZ">不做并行,不管你是多少核cpu.我只有单线程</li>
  <li data-pid="1Hk0DZCj">使用网络资源的时候,不处理UI.甚至就干等,如果网络不稳定.程序就卡住,其实CPU根本没干活</li>
  <li data-pid="hA-gaKHg">写很多大量无效的类型转换/装箱拆箱,声明使用一大堆集合</li>
  <li data-pid="askoaDiP">为了重用代码,同时也懒得优化代码,一件简单的事情,交给一个很复杂的模块处理,说人话就是, 你需要一个螺母,你不直接买一个螺母,你买了一辆汽车,然后把螺母拆下来用,下次我需要一个螺丝,我甚至不用之前买的那辆汽车的螺丝,我再买一辆新的....</li>
  <li data-pid="k5VMgoKC">不管用不用得着,总是使用原始的高清资源.比如用户头像.贪方便.我直接用1080P的图也不压缩...</li>
  <li data-pid="lfVzhmsf">不缓存任何资源,哪怕一个东西反复被使用,依然不停从数据源获取数据...</li>
  <li data-pid="gx-4vkkd">使用完数据不释放.... 比如我买了车取下它的螺母用,但是我车也不处理掉,还放在那做摆设.</li>
  <li data-pid="TLdoHGHh">因为懒得写合理的数据传递.明明要用参数传递的数据一股脑放在全局变量. 拿生活做比喻就是我明明只是和你发私信,但是因为懒得写收件人信息,直接登人民日报...</li>
  <li data-pid="fDDrPLFY">我们拉跨程序员多半知道写的东西垃圾在哪里,他们只是不知道这垃圾啥时候会在实际运行中爆发出问题来.偶尔也不知道这些垃圾组合在一起为啥居然可以运行,更不不知道这些垃圾组合在一起产生哪些新的特性.</li>
 </ul>
 <p data-pid="_xDEJx4B">哦.最重要的,我们拉跨程序员不觉得卡是bug.我们只觉得报错/崩溃/错误的数据展示是BUG...</p>

 <p data-pid="xhvCVNvH">2022.7.10 更新，我说怎么有人来骂我骗人炒股，问题又上热榜了。我随便说几句，那些在华为batm，月入几狗的，可以去拼，去996，但是还有很多在普通公司鬼混的程序员，我说难听一点，35岁，我们这些破程序员竞争力都要下降，各位程序员好好想想，10年前学的技术还有多少有用，业务经验架构倒是有用，但是绝大多数普通人根本进不去那些真的有架构师的公司。在小破公司996有屁用，加班费都不一定有，公司业务好就涨点薪水，最近行情差我那些老同事日子也难过。我自己在前一家公司拼了7年，然后恋爱了，不肯加班了，因为相对薪水好点就被各种搞，我不是让人去股市追涨杀跌做韭菜，平时学一些金融知识，了解一些世界变化，到股市里选一些你觉得未来中国能崛起的公司，在低估值的时候投资，没让你们做敢死队。我过去一年半不到收益90%左右根本没买多少个股，也没有高频交易，也没有盯盘做T。真就是结合一下世界经济的现状，国家产业的优势，国家战略买而已，也没说买到什么很好的位置。也劝大家不要搞什么追涨杀跌。这不叫投机，叫分享国家发展机遇，就像现在房产不行了，但是过去20年房子涨了多少。但是有兴趣的人，可以看看富时a50那些中国龙头过去20年涨了多少，少则几倍多则几十倍比房价还离谱，不要被那些叫上证3000点玩了10年叫骂的骗了，一个程序员，最基本的逻辑能力有吧，能颅内做点推演吧（否则真就全程单步debug？)。未来中国什么行业好什么行业差能有判断吧。年轻的时候拿点闲钱投资，你该上班上班，过几年收益也是支持国家产业升级呀。搞得我多缺德似的。平时多了解世界，多看点书，然后别看太多荐股的，教你搞短线的，你还得上班呢对吧。我过去很长时间家里人都做股票，但是我不屑一顾，觉得程序员工资也还行，也会稳稳上涨，但是等到奔35去了，年纪大了就知道肝不动了，是后悔没有早一点学着存钱学着理财的。我工作这么多年，觉得能坚持做程序的，不管是不是优秀，人普遍都没啥坏心思都是憨憨，所以我知道我们程序员什么勾心斗角，钻营马屁那是多半不太行的。估计就是脑子普遍还行，所以建议大家可以花点心思了解下，基本的东西没那么复杂，但是可以一点闲钱建立习惯，知道自己是不是真的合适。评论我也不处理了，随便吧。</p>
 <p data-pid="pWX_WZvS">2021.11.19 更新，最近还是有人点赞，有点小离谱，这问题生命力真强。</p>
 <p data-pid="R7Ur_NE6">作为彩 大龄程序员，给后辈一个建议，就是多学，但是学精就算了，我等悟性学不精，学得精也很难遇到精密要求自己的自己和要求自己的岗位/公司。多学新东西，就可以去新公司混/或者混到新兴岗位上。</p>
 <p data-pid="_tDcyspZ">目前我在用xamarin搞app 应用。搞得屎一样，好在用户量很少，企业内部用用，也能对付。算是独立开发者，比较自由。可以决定自由的时间是用来学习/游戏/股票。就是觉悟得太晚了。现在房价都不怎么涨，后辈如果现在收入不错，赶紧开始理财吧，不要像我这么大年纪才开始搞。</p>
 <p data-pid="mhaO1hoh">不要脸的说一句，咱们在搞程序里面的，不见得算脑子好用的，但是在搞股票的里面，咱们一定是好用的，程序员不要内卷了，去买股票吧。我今年3月份开始炒股收益率都40%多了</p>

 <p data-pid="sH0xoTev">一天下来，惊呆了，这么多赞，看来获赞的主要原因还是达成共鸣了。</p>
 <p data-pid="xan14TEW">我干程序员也差不多10年了.小学就接触了一些VB,后来工作就发现,很多时候确实工作上很难都高标准要求自己.所以很佩服那些业余时间还会自己做做项目的人. 自己经常项目初期搭个讲究的框架,到开发中期,发现原来设计有点不合理.开始重构,然后一天过去发现业务功能非但没有增加,甚至还制造了更多待优化的项目. 随着时间推移,项目进度就更紧张了.越往后就越随意,再往后就是破窗效应了.一旦你容忍了几个自己项目的白痴代码,接下来就开始放飞自我了. 这就有点像我刚追女朋友的时候,我每天早期刷牙洗脸刮胡子.到同居后,我就要女朋友逼我洗澡了... 现在十几年职业生涯过去.发现所有的技术进步无非集中在:</p>
 <ul>
  <li data-pid="dCOuBubt">经验积累,掌握了更多更快的更容易维护开发的手段.虽然这些手段的性能不见得更好.</li>
  <li data-pid="Gzvh4diu">即使程序未必比过去跑得快,但是程序出问题的时候的bug比过去容易显示了.</li>
  <li data-pid="G2mncq3k">进入职业的中期,深感老板也不在乎的技术,他们脑子都是业务,而你做出自己有成就感的设计提高开发效率的成就感在老板那边甚至你无法和他分享快乐. 最后逼着自己向老板兜售业务.</li>
  <li data-pid="jsiLKdvI">所有的技能居然都来自新业务. 比如搞xamarin 因为我需要做ios了.仅此而已.</li>
 </ul>
 <p data-pid="h62jmptZ">知乎上很多微软 谷歌 BAT 的大牛.他们看看着很潇洒的完成工作. 业余还能发发知乎,带大家逛,甚至做几个牛逼的轮子让大家瞻仰,但是我知道更多的程序员都是我这样平平无奇的人,在大牛眼中可能只是码农码畜,把网络上复制的代码组合加点业务就是我们的工作写照了.但这真的是多数程序员的工作, 没成就更无聊.</p>
 <p data-pid="puPstETg">看到评论大家开心的血压拉满,能给大家带来一点快乐. 或许是我给程序员们带来过的最大价值了.</p>
