 <p data-pid="NvL78QiH">上图中，你可以看到，CPU 里有<b>两部分</b>：</p>
 <ul>
  <li data-pid="kBEI0Sf0">一部分叫做<b> Control Unit</b>，负责控制。比如计数器，指令寄存，等等。</li>
  <li data-pid="es3NG_pT">一部分叫做<b> Logic Unit</b>，负责运算。比如加法器，累加器，等等。有些也会把 ‘ 数据总线’ 归于逻辑运算单元里。</li>
  <li data-pid="zSaOs9Xj">其实 CPU里还有存储单元等其他单元，比如 L1，L2，L3 Cache 等等。</li>
 </ul>
 <p data-pid="XTkRpq6H">输入设备就是 鼠标，键盘，触控板，触摸屏等， 用户通过这些东西对电脑进行输入指令，CPU识别指令后进行运算，进行输出，一般输出给了屏幕。 在这里举一个例子：用户通过触摸屏，对手机进行操作，比如说点击播放音乐，CPU接收到指令后，命令 DAC 解码，扬声器接收到音频信号进行播放，屏幕显示歌词。</p>
 <p data-pid="W9B0OHg1">CPU 就类似于一个管理者，他不去做 ‘苦力’ 活，让更专业的人去做更专业的事。这样就能最大程度上的提高了 整个系统的效率。CPU 虽然叫做 中央处理器，所以它<b>不仅仅是处理</b>，而是对一些更基本的处理器进行<b>控制</b>：</p>
 <p data-pid="x3JaZSbK">‘苦力活‘指的是 CPU 并没有驱动大电流的能力，比如安装好‘驱动软件‘之后，CPU就可以和显示屏上的处理器就行沟通，显示屏上的处理器再来控制显示屏的‘驱动电路’ （Driver），驱动电路驱动显示屏显示图案。在工业上，苦力活用 PLC<sup data-text="可编程逻辑控制器" data-url="https://zh.wikipedia.org/wiki/可编程逻辑控制器" data-draft-node="inline" data-draft-type="reference" data-numero="1">[1]</sup>来控制。一般的，PLC 会受到更上层 CPU 的控制。</p>
 <p data-pid="w7a-ygUN">更为专业的工作指的是：比如图形处理会交给 GPU <sup data-text="GPU" data-url="https://zh.wikipedia.org/wiki/%E5%9C%96%E5%BD%A2%E8%99%95%E7%90%86%E5%99%A8" data-draft-node="inline" data-draft-type="reference" data-numero="2">[2]</sup>来做；音频的解码交给‘声卡’ <sup data-text="声卡" data-url="https://zh.wikipedia.org/wiki/%E5%A3%B0%E5%8D%A1" data-draft-node="inline" data-draft-type="reference" data-numero="3">[3]</sup>AD/DA； 更为高级的音频处理，比如说‘降噪耳机’ 就需要一种叫做 DSP<sup data-text="數位訊號處理器" data-url="https://zh.wikipedia.org/wiki/數位訊號處理器" data-draft-node="inline" data-draft-type="reference" data-numero="4">[4]</sup>进行处理；在为复杂一些的处理，比如说苹果笔记本上的触摸板会有一个专门识别触摸的芯片，叫 单片机<sup data-text="单片机" data-url="https://zh.wikipedia.org/wiki/单片机" data-draft-node="inline" data-draft-type="reference" data-numero="5">[5]</sup>。新的 MAC PRO 加入了 FPGA <sup data-text="现场可编程逻辑门阵列" data-url="https://zh.wikipedia.org/wiki/现场可编程逻辑门阵列" data-draft-node="inline" data-draft-type="reference" data-numero="6">[6]</sup>视频剪辑卡，可以做到 3条8K 视频同时剪辑。</p>
 <p class="ztext-empty-paragraph"><br></p>
 <p data-pid="jculgLlp">（1.1版本补充内容，等有空后会后续补充）</p>
 <p data-pid="6LQJWfZd">computer architecture 计算机的架构<sup data-text="计算机架构" data-url="https://zh.wikipedia.org/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%B3%BB%E7%BB%9F%E7%BB%93%E6%9E%84" data-draft-node="inline" data-draft-type="reference" data-numero="7">[7]</sup>（ISA）是一个非常宽泛的概念：</p>

 <h2>3 CPU 的外形，接口，封装<sup data-text="/Integrated_circuit_packaging" data-url="https://en.wikipedia.org/wiki/Integrated_circuit_packaging" data-draft-node="inline" data-draft-type="reference" data-numero="14">[14]</sup></h2>
 <h2>3.1 封装</h2>
 <p data-pid="PH_uGIZi">整个芯片其实非常非常小，我们外界能看到的产品是对他进行封装好之后的效果，封装当然是主要为了保护芯片，但还有一个很重要的作用就是为了更好的和外界进行沟通（交换信息）。随着封装技术的发展，芯片引脚的密度越来越大，芯片集成程度越来越高。我放几张图，让大家看一下科技的进步把：</p>
 <p data-pid="WIVWevpT">最经典的封装方法：Dual in-line package，DIP（双列直插封装）<sup data-text="Dual_in-line_package" data-url="https://en.wikipedia.org/wiki/Dual_in-line_package" data-draft-node="inline" data-draft-type="reference" data-numero="15">[15]</sup>，学过数电的都接触过这种封装的芯片，这种芯片一般都是 少引脚的。</p>
 <figure data-size="normal">
  <img src="https://picx.zhimg.com/50/v2-1b6fe844f3eb0dabb9303e2a40255e5f_720w.jpg?source=1940ef5c" data-size="normal" data-rawwidth="632" data-rawheight="545" data-original-token="v2-d3dfcb7f8d80f9467f1e22a94edaed64" data-default-watermark-src="https://picx.zhimg.com/50/v2-aee05d6ff97bf819ec30cf8b82f7d8f1_720w.jpg?source=1940ef5c" class="origin_image zh-lightbox-thumb" width="632" data-original="https://pica.zhimg.com/v2-1b6fe844f3eb0dabb9303e2a40255e5f_r.jpg?source=1940ef5c">
  <figcaption>
   Dual in-line package，这也是早期的芯片封装方式
  </figcaption>
 </figure>
 <p data-pid="1yXfoaqP">芯片引脚插入电路，和这种封装配套的技术叫做：“Through-hole technology“<sup data-text="Through-hole_technology" data-url="https://en.wikipedia.org/wiki/Through-hole_technology" data-draft-node="inline" data-draft-type="reference" data-numero="16">[16]</sup>当然这个电路穿孔技术在集成电路里还有另一层意思，比如 3D NAND RAM 里面的穿层技术（这个讲到 Flash 的时候会讲）</p>
 <p data-pid="KswMobTR">DIP 封装是不能满足芯片引脚数目的提高的，此时就有了密度更大的封装方式。PC CPU 里，目前最为复杂的封装方法就是这样的：i9-9900K FCLGA-1151 (LGA) 封装方法<sup data-text="lga-1151" data-url="https://en.wikichip.org/wiki/intel/packages/lga-1151" data-draft-node="inline" data-draft-type="reference" data-numero="17">[17]</sup>，每一个金黄色的点，都是 CPU 和外界进行联系的方式，每一个引脚的功能和使用方法可以查 data sheet 。</p>
 <figure data-size="normal">
  <img src="https://pic1.zhimg.com/50/v2-842a70b0488e21545316cf2cd12f1c34_720w.jpg?source=1940ef5c" data-size="normal" data-rawwidth="600" data-rawheight="400" data-original-token="v2-386be388fc8feaa341f586de9a27da31" data-default-watermark-src="https://pic1.zhimg.com/50/v2-8254f109bd17dda158fa50c3ef23d99b_720w.jpg?source=1940ef5c" class="origin_image zh-lightbox-thumb" width="600" data-original="https://pic1.zhimg.com/v2-842a70b0488e21545316cf2cd12f1c34_r.jpg?source=1940ef5c">
  <figcaption>
   Intel core i9-9900K FCLGA-1151 (LGA)
  </figcaption>
 </figure>
 <p data-pid="0owT2wAA">随着系统的设计复杂，之前的封装结构已经不能满足了，于是就有了更为复杂的封装结构：System on a chip，SoC<sup data-text="System_on_a_chip" data-url="https://en.wikipedia.org/wiki/System_on_a_chip" data-draft-node="inline" data-draft-type="reference" data-numero="18">[18]</sup>，Soc 我会在下一小节的 芯片结构里面用 A12X <sup data-text="Apple-designed_processors" data-url="https://en.wikipedia.org/wiki/Apple-designed_processors" data-draft-node="inline" data-draft-type="reference" data-numero="10">[10]</sup>作为解释。追求更高电路密度的方法就是 Package on package ‘叠电路’<sup data-text="Package_on_package" data-url="https://en.wikipedia.org/wiki/Package_on_package" data-draft-node="inline" data-draft-type="reference" data-numero="19">[19]</sup>，但是由于散热问题反而效果并不特别好，举例就是 iPhone X 的双层电路板结构<sup data-text="高难度iPhone X双层封装主板，现有简单高效的维修方案" data-url="https://zhuanlan.zhihu.com/p/42923962" data-draft-node="inline" data-draft-type="reference" data-numero="20">[20]</sup>。而目前追求电路复杂程度的极致，叫做 System in package， SiP<sup data-text="System_in_package" data-url="https://en.wikipedia.org/wiki/System_in_package" data-draft-node="inline" data-draft-type="reference" data-numero="21">[21]</sup>，把整块电路版想办法做到一起，比如 Apple watch 4 的 S4 芯片<sup data-text="pple+Watch+Series+4" data-url="https://zh.ifixit.com/Guide/Apple+Watch+Series+4++拆解/113044" data-draft-node="inline" data-draft-type="reference" data-numero="22">[22]</sup>。</p>

 <h2>6.3 光刻印刷</h2>
 <p data-pid="7ZSvJ-r_">现在最先进的光刻机用的光是‘极紫光 EUV’ （波长 小于 紫光（193nm）），其实光刻印刷是一件非常神奇一的事情，在我看来最迷人的是这两件：</p>
 <ul>
  <li data-pid="EiGm31tJ">光印原理与流程，电路是怎么一层层的印刷上去的</li>

 <h2>6.4 下脚料与成本</h2>
 <p data-pid="Q4YzgU5r">看到这里，你会很自然地想到，从圆形的 silicon wafer 上切正方形（长方形），那么边角肯定是且不出来的。对，没错，你放大看上面的图就会知道，几乎是wafer 边上一圈全都是废了的。而且这个问题会随着我们需要生产的芯片的面积增大而更严重（里面芯片面积大了，边边废的更多）</p>
 <p data-pid="G87xCWPW">所以<b>不是说生产芯片的厂商不想生产更大的芯片，而是说成本会急剧上升</b>，现在这么做是不经济实惠的。</p>

 <p class="ztext-empty-paragraph"><br></p>
 <p data-pid="y981xIRi">芯片产业虽然是最高科技，但是它本质还是‘产业’是以盈利为目的的。芯片制造与设计要考虑的非常重的一部分就是如何减少成本，而上面的晶元的问题只是冰山一角，真正的 cost 的大头根本不是晶元，生产过程中一个 mask 就要 millions $来计算，所有外行觉得买个光刻机再加一个设计团队不就能生产芯片了？？</p>
 <p data-pid="Gkd8q9L-">真的并不是，它就算花重金做出来，它没有‘经济利益’，是个赔本赚吆喝的过程。</p>
 <h2>6.5 芯片的‘体质’ Process variation<sup data-text="Process_variation" data-url="https://en.wikipedia.org/wiki/Process_variation_(semiconductor)" data-draft-node="inline" data-draft-type="reference" data-numero="54">[54]</sup> 以及市场细分</h2>
 <p data-pid="GtpQvQby">OK，6.4提到了芯片生产商其实是以经济利益为驱动的，但是芯片生产不是生产‘时尚快消品’，不是说要生产出来不同的型号，不同主频，不同缓存的 CPU 来进行‘市场细分’，以剥削消费者。事实上，芯片都是生产出来之后，根据体质的好坏再进行分类包装的。比如：</p>
 <ul>
  <li data-pid="KPL7frS4">生产一个6核 i7，发现坏了一个核心，这时候就直接屏蔽掉两个核心变成 4核i7卖；</li>
  <li data-pid="TfuophAZ">目的是生产16MiB 的 L3 cache，但是坏了2M，那么就在屏蔽一点，按 12Mib 来卖；</li>
  <li data-pid="qCCvHUBP">设计主频是 2.6GHz 的，由于‘体质‘不好，达不到2.6的 就按2.4卖，再不好的就按2.2来卖。</li>
 </ul>
 <p data-pid="isbt90OV">更详细的请戳这里：<a href="https://www.zhihu.com/question/24622588" class="internal">同型号同款 CPU 为什么会有「体质」之分？</a><sup data-text="同型号同款 CPU 为什么会有「体质」之分？" data-url="https://www.zhihu.com/question/24622588" data-draft-node="inline" data-draft-type="reference" data-numero="55">[55]</sup></p>
 <p data-pid="muLuQUPe">我还想补充一下：“wafer 边缘的芯片更容易体质不好，中心的体质会相对好的比例多一些”，这个是有争议性的。<sup data-text="越靠近晶圆圆心，切割出来的 CPU 体质（质量）更好吗？" data-url="https://www.zhihu.com/question/57046428" data-draft-node="inline" data-draft-type="reference" data-numero="56">[56]</sup><sup data-text="Semiconductor devices face many hazards before and after manufacturing that can cause them to fail prematurely." data-url="https://semiengineering.com/why-chips-die/" data-draft-node="inline" data-draft-type="reference" data-numero="57">[57]</sup>不过有一件事就是现代芯片设计过程，考虑到了大量的可能会出现偏差的过程，在设计过程中尽量的避免了。</p>
 <p class="ztext-empty-paragraph"><br></p>
 <p data-pid="1kqBN3Ej">除了上述市场细分之外，苹果和 Intel 有深度合作的排他协议，能拿到最上乘品质的芯片，其他厂商再分中上乘的品质芯片，最后散装的一般都是最差的。mac 里的CPU 是没有散装卖的，虽然很多人会说苹果特供芯片就是普通芯片贴牌，其实准确的说是专门质检过的。</p>
 <h2>7 CPU 的历史以及未来发展方向</h2>
 <p data-pid="6c2h4mw1">1946年，人类历史上第一台由电子管组成的电脑 ENIAC <sup data-text="ENIAC" data-url="https://zh.wikipedia.org/wiki/電子數值積分計算機" data-draft-node="inline" data-draft-type="reference" data-numero="58">[58]</sup>，但是你们肯定听说过它占地多么大，需要多么大的电能；现代电路的发展其实是从 1947年 bell 实验室发明的第一个晶体管开始算起<sup data-text="晶体管" data-url="https://zh.wikipedia.org/wiki/晶体管" data-draft-node="inline" data-draft-type="reference" data-numero="59">[59]</sup>，更准确的说是1960年的场效应管 FET<sup data-text="场效应管" data-url="https://zh.wikipedia.org/wiki/场效应管" data-draft-node="inline" data-draft-type="reference" data-numero="60">[60]</sup>。</p>

  场效应管就是第六章里提到的最基本单元，但由于半导体物理实在是复杂，我并不想多去阐述具体运作原理，大家简单记得电路里最基本的东西就这个：

 <p data-pid="8a4rBkMY">然而到奔腾四<sup data-text="List_of_CPU_power_dissipation_figures" data-url="https://en.wikipedia.org/wiki/List_of_CPU_power_dissipation_figures" data-draft-node="inline" data-draft-type="reference" data-numero="61">[61]</sup>后，CPU 的主频上限已经被卡住了。原因是‘现代’集成电路的能效比其实超级低，<b>大概1%</b>，就是说大部分供电都会转化成热量散失掉了，只有1% 的能量变成了我们运算用的。当我第一次接触各种‘漏电流’的时候，我都对这个世界充满了绝望，what？整个芯片所有地方都在漏，没有不漏的。。。</p>
 <p data-pid="ld-QZrI1">由于能效比这么低，造成的结果就是CPU 非常的热，为了不让他烧坏，<b>CPU 有非常夸张的散热方式，一个大风扇的风冷，或者液冷（水冷）</b>。如果用液冷给他降温，奔腾四就能超频到 5GHz ；但是根本不经济实惠，所以从此集成电路在追求更高的<b>能效比</b>。但绝对不是简简单单的追求能效比，还有诸多考虑的问题。</p>

 <p data-pid="IoXyTCpU">寄存器懒得讲了，感兴趣自己看维基百科<sup data-text="Shift register" data-url="https://en.wikipedia.org/wiki/Shift_register" data-draft-node="inline" data-draft-type="reference" data-numero="65">[65]</sup>吧，DRAM 在结构上是所有存储单元里最简单的<sup data-text="Dynamic_random-access_memory" data-url="https://en.wikipedia.org/wiki/Dynamic_random-access_memory" data-draft-node="inline" data-draft-type="reference" data-numero="66">[66]</sup>：</p>

 <h2>3 TDP，主频，以及超频</h2>
 <p data-pid="sThaUDQ_">3.1 TDP <sup data-text="TDP、TBP、SDP和ACP的区别是什么？" data-url="https://www.zhihu.com/question/67429706/answer/252708550" data-draft-node="inline" data-draft-type="reference" data-numero="83">[83]</sup><sup data-text="TDP是CPU的功耗吗？TDP是固定不变的吗？" data-url="https://zhuanlan.zhihu.com/p/51145563" data-draft-node="inline" data-draft-type="reference" data-numero="84">[84]</sup></p>
 <blockquote data-pid="uvaXKdFv">
  Thermal Design Power (TDP) represents the average power, in watts, the processor dissipates when operating at Base Frequency with all cores active under an Intel-defined, high-complexity workload. Refer to Datasheet for thermal solution requirements.
 </blockquote>
 <p data-pid="blC9KM1_">i9-9900K 的TDP=95W，但是官网是不提供具体损耗的，想知道细节只有前两个方法：1，查datasheet（这个需要问intel 要，而且普通人看不懂）；2，做测试<sup data-text="Intel Core i9-9900K 9th Gen CPU Review: Fastest Gaming Processor Ever" data-url="https://www.tomshardware.com/reviews/intel-core-i9-9900k-9th-gen-cpu,5847-11.html" data-draft-node="inline" data-draft-type="reference" data-numero="85">[85]</sup>，发现极限瞬时功率已经到204W ；3，功率计算器，比如 MSI 提供的<sup data-text="power-supply-calculato" data-url="https://us.msi.com/power-supply-calculator/" data-draft-node="inline" data-draft-type="reference" data-numero="86">[86]</sup>（TDP）。</p>
 <p data-pid="Q2QoYPqZ">而其实现在的芯片都在追求能耗比，即 更小的功率有更好的效率。</p>
 <p data-pid="yfi8tvPV">3.2 瞬时超频的Turbo boost <sup data-text="turbo_boost_technology" data-url="https://en.wikichip.org/wiki/intel/turbo_boost_technology" data-draft-node="inline" data-draft-type="reference" data-numero="87">[87]</sup></p>
 <p data-pid="K1fcYlnc">对于普通用户（非游戏玩家，非专业用户）来说，workload 并不是稳定的，大部分workload 都是非常平稳（低的），在打开软件的瞬间才大。所以瞬时超频就能非常‘聪明’的解决这个问题。</p>
 <p data-pid="iKm071Z3">3.3 低压处理器</p>
 <p data-pid="f1uzaOKF"><b>比如</b>超级本上，我们就限定 <b>功率 TDP = 15W 的基础上，CPU 运行速度越快越好！</b></p>
 <p data-pid="2c0hKU1e">芯片里的功耗，最大的功耗就是‘<b>漏</b>’掉了，如果你深入研究会发现，几乎所有地方都在漏，没有不漏的，生产工艺/设计工艺里，有很大一部分都在研究如何减少‘漏电‘的情况。不过先抛开怎么减少的漏电，我们先直观的感受一下 CPU 的功耗是什么<sup data-text="CPU_power_dissipation" data-url="https://en.wikipedia.org/wiki/CPU_power_dissipation" data-draft-node="inline" data-draft-type="reference" data-numero="88">[88]</sup>：</p>
 <p data-pid="hQ8FLzyZ"><img src="https://www.zhihu.com/equation?tex=P_%7Bdym%7D+%3D+CV%5E%7B2%7Df" alt="P_{dym} = CV^{2}f" eeimg="1"> ，<i>C </i>is capacitance, <i>f </i>is frequency, and <i>V </i>is voltage。</p>
 <p data-pid="c9nVX1ZR">这个公式特别好理解，<b>CPU 的功耗和 电路运行的速度成正比，和电路运行的电压的平方成正比。</b>所以想要减少CPU 的功率，就要想办法降低这三个值。</p>
 <p data-pid="l4qFf8-Q">非常的巧：如果我们把 门做小， 电容 C 会随之变小（电路设计里，怎么制作大 C 是个非常大的问题）； f 我们想保持不变，减少电压 V 就是最好的办法。<b>这就是你所见到的那些‘超级本’ 用的 “低压处理器“的原因。</b></p>
 <p data-pid="iXLQd2h2">如果你还感兴趣，可以继续看这个问题：<a href="https://www.zhihu.com/question/303037255/answer/558630384" class="internal">为什么计算机芯片大都采用低电压大电流的供电方案？</a><sup data-text="为什么计算机芯片大都采用低电压大电流的供电方案" data-url="https://www.zhihu.com/question/303037255/answer/558630384" data-draft-node="inline" data-draft-type="reference" data-numero="89">[89]</sup></p>
 <p data-pid="jEEgPBsj">3.4 为什么低电压就很难高功率，为什么超频需要高电压？</p>
 <p data-pid="8D1i5ece">有人告诉你说，<b>调高电压，就能超频</b>，但是没有告诉你为什么；在1.2 章节里提到了，我们希望功率低，但频率高，那么唯一可行的办法就是降低 C 以及 降低 V。</p>
 <p data-pid="740cjWV8">你可能会想，V 为什么不能降低？<b>加载到门极上的电压低而运行在高频率（不断的开关）的的时候，CPU 会出错</b>。如果想要‘感性的认识’而不是直接拿过来公式给你展示出来的话就是：‘我们把电压类比成关门的时候用的力气，高电压就好像是用力关门，关门会关的很紧，而低电压的时候（门级电压不够高），关门关不紧，这个时候就会让<b>电子泄露</b>，导致<b> 0/1 混乱</b>。<b>（当然实际上，门级电压是怎么影响运行速度的不是这么计算的，在这里是为了让人更好理解。）</b></p>
 <p data-pid="c2rcQSFR">同样的，我们高电压的时候，CPU 就更不容易出错，就可以在更高的速率上运行。这也就是俗称的‘体质’，我借用一下别人的图<sup data-text="同型号同款 CPU 为什么会有「体质」之分？" data-url="https://www.zhihu.com/question/24622588" data-draft-node="inline" data-draft-type="reference" data-numero="55">[55]</sup>来让你更好的理解一下：</p>
 <figure data-size="normal">
  <img src="https://picx.zhimg.com/50/v2-d7ea58dc5baf6bb3b4f85db1b07cfb09_720w.jpg?source=1940ef5c" data-caption="" data-size="normal" data-rawwidth="519" data-rawheight="399" data-original-token="v2-215e8836ad1d04509e1da6ced5a3dad6" data-default-watermark-src="https://pic1.zhimg.com/50/v2-ac7c765351fed4e9249a9cd552977fc8_720w.jpg?source=1940ef5c" class="origin_image zh-lightbox-thumb" width="519" data-original="https://pica.zhimg.com/v2-d7ea58dc5baf6bb3b4f85db1b07cfb09_r.jpg?source=1940ef5c">
 </figure>
 <p data-pid="-3s0SL72">横坐标是 CPU 运行的频率，纵坐标是CPU 运行电压，绿色表示 CPU正常工作，红色表示 CPU 出现错误。从图上可以看清楚，为了保证CPU 处于正常工作状态下更高的频率，就需要更高的电压。</p>
 <p data-pid="CS2mMkbH">所有些人说‘调高电压就会超频’ 是把<b>因果弄反</b>，不是说我们调高了电压 CPU 就会更高的主频，而是说，<b>想维持CPU 正常工作的前提下，更高的主频需要更高的电压。</b></p>
