 <h2>本答案的核心思路：越简单越好。</h2>
 <p data-pid="5dO4NcFq">这些年软硬件及个人对NAS的认知也在不断更新，期间有一些新的想法方便大家避坑。</p>
 <p data-pid="dwvEbzPK">折腾NAS的四年中，本人换来换去用过方案很多，黑群晖、Linux、Windows、OS X、各种虚拟机混搭等等，现在感觉搭建家用NAS的目的是简单易用，而不是把自己搞到身心俱疲。</p>
 <hr>
 <h2>首先 —— 你是否真的需要一台NAS：</h2>
 <ul>
  <li data-pid="uevBlfbn">如果您需要的只是挂机下载、SMB局域网共享文件：PC或路由器+移动硬盘；</li>
  <li data-pid="UeWBm2YL">如果在1的基础上还需要作视频服务器：PC，核显即可；</li>
  <li data-pid="p6G2TQ2P">如果你需要<b>手机相册同步</b>、<b>外网调用/分享个人云上的文件</b>，并且要有<b>成熟的APP支持</b>，那么你需要一台NAS。</li>
 </ul>
 <p data-pid="ox6NZ8qb">NAS：私人云相册、异地(外网)分享、各种成熟APP。</p>
 <p data-pid="R_qK6AH2">注：目前windows/linux上没有完备的私人相册方案(服务器端+客户端app)，NextCloud的相册功能比起群晖/威联通来差距较大。</p>
 <p class="ztext-empty-paragraph"><br></p>
 <h3>重复一下：私人相册基本只能选成品NAS。</h3>
 <hr>
 <h2>方案选型太长不看版：（以时间成本排序）</h2>
 <ul>
  <li data-pid="g3vmOwJI">图省心开箱即用、客户端完备的：群晖/威联通；</li>
  <li data-pid="lU8_F2fy">仅需内网文件共享：Windows（简单，无学习成本），配合Plex/Emby可实现外网视频分享；</li>
  <li data-pid="ZCdek_5i">需要外网文件分享的：OMV / UNRAID，自带基本功能，其他模块Docker安装；</li>
  <li data-pid="9svppd7d">超清/原盘影视下载收集狂：UNRAID，可随时添加硬盘扩容；</li>
  <li data-pid="7sgix0Tm">全功能个人云盘：Linux（高定制化，且Nextcloud/Seafile只有linux版）；</li>
  <li data-pid="zJ_jqfWi">蛋疼非要在小鸡下跑NAS的痴汉：Proxmox VE（家用主板不建议ESXi）。</li>
 </ul>
 <hr>
 <h2>基本概念</h2>
 <p data-pid="rNuCRRbC">NAS即网络附加存储(Network Attached Storage)，通过网络提供数据访问服务。</p>
 <p data-pid="Xv4JLX1T">NAS产品逐渐被大众熟知的今天，相当多的用户会认为此物是个人/家用产品，而实际上这货面向的并不是家庭用户，这也是为什么NAS系统都有RAID模块的原因。</p>
 <blockquote data-pid="gnyVyHdI">
  注：本答案面向有外部访问/远程控制需求的用户，如仅做内网备份/文件共享/视频播放的朋友们装个windows最省心，顶多通过docker增加TimeMachine备份功能。
 </blockquote>
 <hr>
 <h2>基本需求</h2>
 <ol>
  <li data-pid="DMchqx8d"><b>硬件——NAS硬件需要满足以下几点：</b></li>
 </ol>
 <ul>
  <li data-pid="6fv_tFjy">稳定性：24x7稳定无故障运行；</li>
  <li data-pid="t9LX7HNs">兼容性：硬件对操作系统兼容性良好(系统电源管理、硬盘控制器兼容性等)；</li>
  <li data-pid="srlZVNSd">体积小巧：占地面积小，便于放置；</li>
  <li data-pid="RZvuohua">节能：7x24小时挂机下载的情况下，动辄一两百瓦的家用电脑显然不适合作NAS；</li>
  <li data-pid="0n-TXxcu">静音：做工优秀，廉价机箱的箱体共振等问题不应该存在；</li>
  <li data-pid="91S7aO_1">性价比：合理价格内兼顾以上6点。</li>
 </ul>
 <p data-pid="QqJC2zfq"><br><b>2. 软件——NAS的功能需求：</b></p>
 <ul>
  <li data-pid="j1I4JNck">须具备远程控制功能如Web面板，方便配置操作；</li>
  <li data-pid="nsN7zTYW">须有移动端app的支持；</li>
  <li data-pid="bwkE1NmN">挂机下载(PT/BT/电驴/各种云盘同步)；</li>
  <li data-pid="q1q6MzXw">媒体服务器(配合机顶盒或本机播放)；</li>
  <li data-pid="aDNkiXju">本地及远程文件共享(个人/家庭云)；</li>
  <li data-pid="3pKciUOx">文件备份(TimeMachine等)；</li>
  <li data-pid="NDpw7FLW">(附加)办公用途(CardDAV Server / Directory Server / Mail Server)；</li>
  <li data-pid="a9xflh9B">(附加)其它用途(DNS Server / 小型网站服务器等)。</li>
 </ul>
 <p data-pid="bgcCnhNK"><b>3.远程控制：</b></p>
 <ul>
  <li data-pid="dW3lTEqy">作为一台NAS，远程控制(包括局域网内)应为刚需；</li>
  <li data-pid="1z16NVgh">远程控制方式包括远程桌面、WebUI两种方式：</li>
  <li data-pid="afXC3jYY">远程桌面即RemoteDesktop、TeamViewer、JumpDesktop等，WebUI即浏览器面板，如群晖的DSM、Linux下的Cockpit/Webmin、Windows的Windows Admin Center等。当然Plex、Aria2也是WebUI的典型方式；</li>
  <li data-pid="aaS0nIoM">个人建议WebUI方式，浏览器内搞定，无需安装客户端软件，手机/Pad皆可控制NAS。</li>
 </ul>
 <p data-pid="7uPSh3Si"><b>其他：</b>部分用户希望NAS具备HTPC功能，本答案已充分考虑。</p>
 <hr>
 <h2>写在前面1：关于DDNS</h2>
 <p data-pid="HmZxhuJq">DDNS是各类远程操作的必需品，简单罗列下本人用过的几种：</p>
 <ul>
  <li data-pid="DLnuG3gA">华硕路由自带asuscomm方案：稳定性尚可，但出现过中断的情况；</li>
  <li data-pid="4gjXBm0L">群晖QuickConnect：仅白群晖用户可用，亮点是自带内网穿透，实属内网用户福音；</li>
  <li data-pid="j2T6dhz6">花生壳/花生棒：尚可，支持各类操作系统平台(路由器/Windows/Linux/群晖)；</li>
  <li data-pid="5mfLFtzO">阿里云DDNS：强烈推荐；</li>
  <li data-pid="PZ4NOY07">腾讯云DDNS：推荐，证书验证不如阿里云方便。</li>
 </ul>
 <h2>写在前面2：关于RAID及备份</h2>
 <ul>
  <li data-pid="652wa0fj">友情提示：家庭用途不建议配置RAID，孱弱的软RAID机制不要指望稳定性；</li>
  <li data-pid="iejA321C">千兆网络带宽只能跑到单盘速度，家用环境RAID0用途不大；</li>
  <li data-pid="AEwIS-vT">RAID5模式下任何文件操作所有盘都参与读写，于家庭用途中甚为蛋疼；</li>
  <li data-pid="v9KLuCB3">RAID5模式下磁盘故障后换盘重建的时间非常长，甚至会有坏第二块盘的可能；</li>
  <li data-pid="TgkkeBMD">数据无价。与其折腾RAID不如冷备份。</li>
  <li data-pid="6r09tdLa">没有BckupPlus或MyBook 8TB不能解决的备份问题，如果不能，那就两块(海淘价格有惊喜)。</li>
 </ul>
 <h2>写在前面3：关于磁盘的选择</h2>
 <figure data-size="normal">
  <img src="https://pica.zhimg.com/50/v2-60164574038d75b22233ca09f1c81367_720w.jpg?source=1940ef5c" data-caption="" data-size="normal" data-rawwidth="636" data-rawheight="97" data-original-token="v2-60164574038d75b22233ca09f1c81367" class="origin_image zh-lightbox-thumb" width="636" data-original="https://pic1.zhimg.com/v2-60164574038d75b22233ca09f1c81367_r.jpg?source=1940ef5c">
 </figure>
 <p data-pid="lgl7Z-_U"><b>不要神化所谓NAS硬盘，本人两块国行4T红盘在2万小时左右都挂了</b>(非24x7工况、家用环境读写不频繁、未进行PT/BT下载)，这货的MTBF可是号称100万小时的。</p>
 <ul>
  <li data-pid="-7kN5dMY">“NAS硬盘”这个概念起于WD红盘，因红盘的大卖，Seagate、Toshiba也跟进推出NAS硬盘，但无论是WD的"NAS Ware"，还是Seagete的“AgileArray”技术，都是在RAID错误校正、减少RAID重建时间等方面的技术；</li>
  <li data-pid="VjypQPuB">至于震动补偿等技术，在4盘以上的多盘系统下才能够充分显现作用；</li>
  <li data-pid="EIFZZV87">与几十上百个客户端同时访问的企业NAS不同，家用环境下磁盘工作强度非常低，日常数据吞吐量极小；</li>
  <li data-pid="rPbVkjQY">既然家用环境下磁盘工作强度低，RAID模式又蛋疼，震动补偿又只算个赠品，所以硬盘选择方面丰俭由人，NAS专用硬盘对于家庭/个人用户并不能算做硬需求。</li>
 </ul>
 <p data-pid="CaIHfyad">综上所述，磁盘方面如下考量：</p>
 <ul>
  <li data-pid="rPuhkfJu">24x7开机 --&gt; 紫盘或酷鹰，建议企业盘；</li>
  <li data-pid="y_-vugco">非24x7开机 --&gt; 普通磁盘；</li>
  <li data-pid="3w2qudvc">需要RAID--&gt; 红盘或酷狼，建议企业盘；</li>
  <li data-pid="VWVNhxxj">如果没有冷/热备份的打算，仍然建议企业盘。</li>
 </ul>
 <p data-pid="vg7V5VEt">(其实我的建议是：要么企业盘，要么桌面盘)</p>
 <h2>写在前面4：关于磁盘格式</h2>
 <ul>
  <li data-pid="vzTQfWmt">功能越强的文件格式速度往往越慢(Btrfs慢，ZFS吃内存)；</li>
  <li data-pid="1dBcyxrG">工作文件夹共享、文件内容频繁更新的磁盘可使用支持回滚操作的Btrfs/ZFS；</li>
  <li data-pid="-cATvxpq">仅作下载/存储用途不需要快照功能，ext4足矣，简单方便速度快。</li>
 </ul>
 <h2>写在前面5：关于黑群晖</h2>
 <p data-pid="3BBjSxaz">目前黑群晖已无法洗白，即便能通过DDNS解决无QuickConnect远程访问，但个人感觉仍然问题多多。大致总结如下：</p>
 <p data-pid="UTKN0mYL">A. 软件方面——功能缺失：</p>
 <ul>
  <li data-pid="_B_VvyyI">系统不能更新/迭代，安全漏洞自然也无法修复；</li>
  <li data-pid="MLdpAcUa">DS Finder无推送(半洗白无解)；</li>
  <li data-pid="ihePv-UX">Chat无消息提醒(半洗白无解)；</li>
  <li data-pid="TEZMO_JU">DS Video无硬解码(半洗白可解)；</li>
  <li data-pid="PZJoWaJd">Virtual Machine Manager无法启动虚拟机(提示内存不足)(半洗白无解)；</li>
  <li data-pid="r6pM64Du">High Availability Manager无法使用(半洗白无解)。</li>
 </ul>
 <p data-pid="4S6rciVy">B. 硬件方面——兼容性问题：</p>
 <ul>
  <li data-pid="JTa1Bse0">DSM6.2的bootloader对网卡驱动支持不全，导致无法安装或升级系统(需换网卡)；</li>
  <li data-pid="8PeNhJp2">由硬件兼容性导致的硬盘无法休眠(视bootloader)；</li>
  <li data-pid="A8CdD5FY">设置定时开机导致BIOS重置(视硬件设备)；</li>
  <li data-pid="cquQ4ViF">磁盘性能低下，如Gen10在DSM6.1下磁盘对拷60M/s (DS3617xs)；</li>
  <li data-pid="BY-ilIFk">J3455-itx DSM6.1磁盘性能极低，DSM6.2无法关机/重启。</li>
 </ul>
 <p data-pid="7YECtWkO">C. 虚拟机方面——只能作为玩具用途：</p>
 <ul>
  <li data-pid="QXy-ncWh">VMware虚拟网卡不能跑满带宽；</li>
  <li data-pid="Hq49lhXH">VMware挂载实体盘会因诸如重装/迁移宿主系统时忘记attribute disk clear readonly等操作导致DSM提示磁盘损毁，好像还无法修复；</li>
  <li data-pid="zCNEOqlH">目前Hyper-V对DSM6及以上黑群晖无解；</li>
  <li data-pid="Mkifv2fR">EXSi不能休眠的来着(虽然不是黑群晖的锅，但家庭用途用EXSi干毛)；</li>
  <li data-pid="Cpe6f2ZL">未直通SATA的PVE/EXSi，黑群晖无法读取S.M.A.R.T信息，而直通的话……话说你都直通了干嘛不实体安装；</li>
  <li data-pid="XEbEW0jK">如果用PVE/EXSi的目的是软路由+黑群晖，不如实体黑群晖的VMM中跑软路由。</li>
 </ul>
 <p data-pid="GYeh9-D0">（如果您一定要用黑群晖，个人建议实体安装）</p>
 <p data-pid="Lb2b_sHt">——虽然黑群晖一张启动优盘搞定安装，但后期的折腾并不比装个Windows少，况且并不是没有群晖的替代方案。</p>
 <p data-pid="RMdEJxVT"><b>小结：数据无价，远离黑群晖。</b></p>

 <p data-pid="2uTJN88o"><b>OS/软件方案B：Linux平台</b></p>
 <p data-pid="EZ-CYgRQ">Linux与Windows相比胜在简便，缺点为需要花点时间学一下基础命令，大致如下4个：</p>
 <ul>
  <li data-pid="aqOd7q3R">ssh：登录NAS；</li>
  <li data-pid="S2HHkMh6">yum/apt-get/dnf：软件包安装/管理（具体命令视你安装的linux发行版）；</li>
  <li data-pid="eDGTuv4M">vim：各种配置文件的编辑（知道-i、:wq即可）；</li>
  <li data-pid="2gAk2swc">git/wget：获取部分文件的常用命令。</li>
 </ul>
 <p data-pid="rvZ79BLT">大致思路：</p>
 <ol>
  <li data-pid="-dZ41a0g">ssh登录进NAS，yum/apt-get/dnf安装cockpit/docker，你的NAS基础系统已构建完毕。</li>
  <li data-pid="fiRXOnPr">Nextcloud、迅雷远程、Aria2、Plex等等全部用docker搞定，省却繁复的命令行配置。</li>
 </ol>
 <p data-pid="Ap7xZJiO"><b>为什么推荐docker：</b>linux各发行版间存在差异，以nextcloud为例，CentOS/ubuntu的基础服务配置不同，甚至ubuntu17下用的好好的nextcloud在系统升级至18后要重新配置。系统迭代、重新安装后都需要较繁复的配置工作，这显然是与本文“简单易用别折腾”的思路相悖的。即便系统升级换代，用docker重装nextcloud+mariadb+onlyoffice也仅需30秒。</p>

 <p data-pid="6Y7tcsP_"><b>2. FreeNAS：非家庭用途</b></p>
 <p data-pid="vIilTKWf">基于FreeBSD。ZFS格式的内存吞噬者称号并非浪得虚名，8G内存起步(且官方强烈建议ECC内存)，在此基础上，官方建议如下配置：</p>
 <ul>
  <li data-pid="D5sbN9gC">磁盘容量：每1T磁盘容量再增加1G内存；</li>
  <li data-pid="BEqCGars">磁盘容量：如需对ZFS数据去重，每TB数据至少5G内存；</li>
  <li data-pid="2sB-jfDC">iSCSI：至少16G内存(低性能)，32G内存(良好性能)；</li>
  <li data-pid="RXOI4t2s">Active Directory：2G内存；</li>
  <li data-pid="d35bC6Ij">Jails：1-4G内存；</li>
  <li data-pid="nKdaSDYp">虚拟机：与分配给VM的内存同等内存容量；</li>
  <li data-pid="1XHjiWp5">同时，官方还建议采用高主频的CPU。</li>
 </ul>
 <p data-pid="U95BtF52">也就是说，如果你的NAS存储了8TB数据，官方建议配置16G的内存。如果你的NAS塞了32TB数据，按官方的建议应该是40G内存，这还是没开ZFS数据去重、iSCSI、ActiveDirectory和虚拟机的情况下！</p>
 <p data-pid="c9eNYGdp">————所以大家应该清楚这货根本不是给家庭用户设计的。</p>
 <p data-pid="JcdLYToe">优点：现代化的UI，各项设置及操作一览无余，PT/BT等功能以插件模式安装(然鹅nextcloud版本老的可以)，自带虚拟机功能。</p>
