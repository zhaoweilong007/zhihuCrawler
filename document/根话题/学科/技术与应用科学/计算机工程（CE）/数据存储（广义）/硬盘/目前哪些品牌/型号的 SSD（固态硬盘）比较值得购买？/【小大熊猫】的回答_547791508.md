 <h2>简略版</h2>
 <h3><b>1.</b>玩游戏买什么固态硬盘？</h3>
 <p data-pid="kILwhZ50">根据各种评测，对于绝大部分游戏（以单机游戏为例，如GTA5、巫师3和古墓丽影11等，网游则影响加载速度上限，因为可能需要等待其他玩家），在其它配置相同的情况下，<b>将游戏放在SATA固态硬盘中的加载速度比将游戏放在机械硬盘中的加载速度快一倍（载入时间短一半），而NVMe固态硬盘的加载速度只比SATA固态硬盘快20%不到（视游戏而定，多数情况感觉不明显）。原因：绝大多数固态硬盘在低队列深度下的4K随机性能差距不大，而且已经能较好地满足主流游戏的需求。在预算一定的情况下，优先选择容量更大的型号。</b></p>
 <p data-pid="MTW0bA8f">对于NVMe固态硬盘，普通用户通常没必要关注其是否搭载外置DRAM缓存，支持HMB技术的主流NVMe固态硬盘性能基本能满足普通家用需求：<a href="https://www.zhihu.com/question/525768248/answer/2429886221" class="internal">HMB技术能多大程度上提高固态硬盘的性能？ - 小大熊猫的回答 - 知乎</a></p>

 <h2>前言</h2>
 <h3><b>1.</b>什么是固态硬盘？</h3>
 <p data-pid="ZALMpIpE">固态硬盘一般指采用NAND闪存（或3D XPoint）作为存储介质的非易失性存储器，与之相对应的概念是机械硬盘（温彻斯特硬盘）。固态硬盘的英文名称是Solid State Disk或者Solid State Drive（即SSD），而机械硬盘一般被称为Hard Disk Drive（即HDD）（光盘驱动器Optical Dsik Drive，软盘驱动器Floppy Disk Drive）（闪存即指Flash芯片）。</p>
 <ul>
  <li data-pid="9zZzEy4y">NAND闪存基于浮栅金属氧化物半导体场效应晶体管（Floating-gate MOSFET，简称浮栅MOSFET或FGMOS），通过修改存储在浮栅中的电荷量来表示数据。<sup data-text="浮栅金属氧化物半导体场效应晶体管 - 维基百科，自由的百科全书" data-url="https://zh.wikipedia.org/wiki/%E6%B5%AE%E6%A0%85%E9%87%91%E5%B1%9E%E6%B0%A7%E5%8C%96%E7%89%A9%E5%8D%8A%E5%AF%BC%E4%BD%93%E5%9C%BA%E6%95%88%E5%BA%94%E6%99%B6%E4%BD%93%E7%AE%A1" data-draft-node="inline" data-draft-type="reference" data-numero="1">[1]</sup>在2D NAND闪存时代，浮栅型结构（Floating Gate）几乎是唯一的选择。在转换到3D NAND闪存时，电荷捕获型结构（Charge Trap）后来居上，成为了主流选择。目前绝大多数NAND闪存制造商生产的3D NAND闪存都基于电荷捕获型结构。<sup data-text="为什么英特尔_美光的3D闪存还在用浮栅型结构？_PCEVA,PC绝对领域,探寻真正的电脑知识" data-url="http://www.pceva.com.cn/article/3751-1.html" data-draft-node="inline" data-draft-type="reference" data-numero="2">[2]</sup></li>
  <li data-pid="ngTw7UX7">NAND闪存的基本架构为Chip/Package-Die-Plane-Block-Page-Cell。</li>
  <li data-pid="y5uiae10">其中Cell为NAND闪存的基本单元，根据一个Cell存储数据的多少，可以将NAND闪存分为SLC、MLC、TLC和QLC四种（分别为1bit、2bit、3bit和4bit）。目前SLC和MLC闪存只有极少数高端产品采用，主流固态硬盘均基于TLC和QLC闪存。</li>
 </ul>
 <p data-pid="jp5R9ZHJ"><a href="https://www.zhihu.com/question/336474160/answer/1430969741" class="internal">如何浅显易懂地解释「闪存」？它的存储原理是什么？ - 小大熊猫的回答 - 知乎</a></p>
 <h3><b>2.</b>固态硬盘有什么特点？</h3>
 <p data-pid="uPRBGd4P">与机械硬盘相比，固态硬盘拥有更快的读写速度（无论是顺序读写还是随机读写）、更高的IOPS（每秒响应次数）以及更低的响应延迟（类比机械硬盘的寻道时间）。由于没有像机械硬盘那样的运动部件，固态硬盘对振动/震动没那么敏感，运行时发出的噪音几乎不可闻。但是，固态硬盘在发生故障或因误操作删除、覆盖数据时恢复难度相对较高。</p>
 <h3><b>3.</b>使用固态硬盘（作为系统盘）是怎样的体验？</h3>
 <p data-pid="_V9xWzFS">与机械硬盘相比，安装在固态硬盘上的操作系统和应用软件的启动、加载时间显著缩短，系统响应速度加快，总体而言使用更加流畅。</p>
 <h3><b>4.</b>固态硬盘有哪些分类？</h3>
 <p data-pid="wkJmOjbE">根据物理外形可以分为2.5英寸、M.2和AIC（独立显卡那样的插卡式）三类。</p>
 <p data-pid="GV60qsG9">根据电气标准可以分为SATA和PCIe两类。</p>
 <p data-pid="Pn7ape8X">根据数据协议可以分为AHCI和NVMe两类。</p>

 <h3><b>5.</b>网上常说的固态硬盘的掉速是指什么？</h3>
 <p data-pid="MiF4Ibzr">由于NAND闪存的特性（无法直接覆写，NAND闪存的最小写入单元是page，最小擦除单元是block，多个page组成一个block，多个block组成一个die），因此在向存有数据（无论数据是否有效）的block写入数据时必须先擦除再写入，表现为读写速度低于预期。通常固态硬盘主控会自动进行垃圾回收和TRIM操作，可在一定程度上缓解此类掉速现象。</p>
 <p data-pid="5A695RTq">此外，几乎所有消费级TLC固态硬盘都采用了动态SLC缓存机制。在连续写入大量数据时，SLC缓存用尽后写入速度会降至TLC原本的水平（闲置一段时间后自动恢复）。</p>
 <h3><b>6.</b>固态硬盘的寿命/耐久度取决于哪些因素？</h3>
 <p data-pid="d2lsSVgk">固态硬盘的可靠性衡量指标有位错误率、数据保持时间、故障率、终身写入量/日负载量、工作和非工作状态下的耐受温度、冲击加速度和振动加速度等。其中终身写入量/日负载量是主要的寿命/耐久度衡量指标。闪存写入量和通电时间也是衡量固态硬盘新旧程度的主要指标。</p>
 <ul>
  <li data-pid="dzJ-3NUu">无法纠正的位错误率（Uncorrectable Bit Error Rate (UBER)）</li>
  <li data-pid="XFDdqv-L">数据保持时间（Data Retention）</li>
  <li data-pid="heIwbjyi">年故障率（Annual failure rate (AFR)）</li>
  <li data-pid="GU7lJ0sQ">平均故障间隔时间（Mean Time Between Failures (MTBF)）</li>
 </ul>
 <p data-pid="lC2NNpYq">数据保持时间（Data Retention）是衡量NAND闪存可靠性的重要指标。</p>
 <p data-pid="RBL6L4AD">由于浮栅MOSFET中本征电场的存在，随着时间流逝产生的电荷泄漏，会逐渐导致NAND闪存中的数据出错甚至无法读取。为此Read Scrub（数据巡检/扫描重写）技术应运而生，通过按照一定的算法对硬盘进行扫描，当发现某个闪存页翻转比特数超过阈值时，对数据进行重写，从而减少ECC无法纠正的位错误。<sup data-text="闪存数据保存期和完整性技术" data-url="http://www.ssdfans.com/?p=90860" data-draft-node="inline" data-draft-type="reference" data-numero="3">[3]</sup>影响数据保持时间的因素有擦写次数、ECC强度、温度和电磁辐射等。<sup data-text="闪存数据保存时间（Nand Flash data Retention）测试方法 - 兵哥的文章 - 知乎" data-url="https://zhuanlan.zhihu.com/p/51638830" data-draft-node="inline" data-draft-type="reference" data-numero="4">[4]</sup></p>
 <p data-pid="9kdRVVBa">由于陷阱辅助隧穿（Trap-assisted tunneling (TAT)）效应，随着擦写次数（编程/擦除周期数，Program/Erase Cycle (P/E Cycle)）的增加，氧化物绝缘层逐渐磨损，NAND闪存的可靠性会逐渐降低（数据保持时间缩短）。<sup data-text="闪存数据保存期和完整性技术" data-url="http://www.ssdfans.com/?p=90860" data-draft-node="inline" data-draft-type="reference" data-numero="3">[3]</sup>升高温度也会使数据保持时间缩短。<sup data-text="闪存数据保存时间（Nand Flash data Retention）测试方法 - 兵哥的文章 - 知乎" data-url="https://zhuanlan.zhihu.com/p/51638830" data-draft-node="inline" data-draft-type="reference" data-numero="4">[4]</sup></p>
 <p data-pid="YEfJyyCb">由于读取干扰（Read Disturb）的存在，即便不进行写入，读取操作也会使相近的存储单元受到影响。主控会计算自上次擦除动作后的读取动作总次数，通过重写相应存储单元中的数据，避免读取干扰错误的发生。<sup data-text="闪存 - 维基百科，自由的百科全书" data-url="https://zh.wikipedia.org/wiki/%E9%97%AA%E5%AD%98" data-draft-node="inline" data-draft-type="reference" data-numero="5">[5]</sup></p>
 <p data-pid="iRtICyOf">当NAND闪存的可靠性下降至厂商设定的阈值时，其生命也就抵达了终点。通常此时固态硬盘的S.M.A.R.T.参数会达到阈值（P/E Cycle达到阈值、坏块数达到阈值或其它耐久度参数达到阈值），引发系统报警，提醒用户更换硬盘。这种情况可以理解为寿终正寝。</p>
 <p data-pid="64RWgnSh">但是固态硬盘也可能在寿终正寝之前发生致命错误导致失去响应无法被系统识别（俗称掉盘）。</p>
 <p data-pid="Zmz3U4um">此外随着固态硬盘存放时间的增加，其故障率逐渐升高，因此应尽可能购买新出厂的产品。</p>
 <h3><b>7.</b>为什么固态硬盘在系统中显示的容量小于标称值？</h3>
 <p data-pid="t0IdynXD">因为其中一部分容量被划分为预留空间（Over-Provisioning），用于存放FTL表和坏块替换等。</p>
 <p data-pid="d2SVl6uk">按照国际标准化组织（以及Windows）的计算方法，1GiB=1024*1024*1024Byte（GiB≠GB），固态硬盘的FLASH实际容量应是如此，但用户可用空间按1GB=1,000,000,000Byte给出，其余部分用作OP对用户不可见。</p>
 <h3>8.如何选购固态硬盘？</h3>
 <p data-pid="2C6sGlZg">在选购硬盘时，应当关注容量、性能和可靠性三个方面。</p>
 <p data-pid="tDUkCgQ1">对于机械硬盘，由于只有寥寥几家制造商，一般考虑盘片记录技术（CMR、SMR和HAMR等，详见匿名知乎用户的回答<a href="https://www.zhihu.com/question/289798339/answer/490319677" class="internal">为什么目前市面上的机械硬盘大都不标注是否使用SMR技术？ - 知乎</a>）、单碟容量、转速、缓存大小、外部接口以及充氦与否，再加上读写速度和响应时间实测值以及硬盘故障率，很容易就能得出大致评价。考虑到机械硬盘行业门槛较高且已经十分成熟，几大机械硬盘制造商的产品并无太大差距，可以直接按照厂商划分的产品线进行选择。</p>
 <p data-pid="vd7BybTD">而对于固态硬盘，不仅参数十分繁杂，仅靠分析这些参数并不能准确地推测出硬盘的实际性能，而且固态硬盘在不同工作条件下表现出的性能也有很大差异。至于固态硬盘的可靠性，也缺乏大量数据支持（与机械硬盘相比）。但是仍然建议用户优先购买TLC固态硬盘而不是QLC固态硬盘。</p>
 <p data-pid="jdMuoQBH">建议普通用户尽量选择一线品牌的主流型号，因为一线品牌的产品销量相对较大，它们不仅经受了大量用户的考验，而且即使变更方案，也很快会由用户反映体现出来。此外一线厂商的售后服务通常也更好（比如在店保之外能个人送保）。</p>
 <p data-pid="EHKlMsST">在消费级固态硬盘制造商里，主要一线品牌有三星、英特尔（现已出售给给海力士）、闪迪+西数、东芝+浦科特和美光（镁光）+英睿达。这些制造商大多具有自行生产NAND闪存的能力，某些品牌还能自研主控（如三星）。</p>
 <p data-pid="Ef88yw20">固态硬盘的主要部件有主控、DRAM缓存和闪存芯片。其中DRAM缓存为非必需，主控可能和闪存芯片集成在一起（比如某些M.2 2230规格的固态硬盘）。</p>
 <ul>
  <li data-pid="CjsIFkXN">常见主控制造商：三星电子(Samsung)、美满电子(Marvell)、群联电子(Phison)、慧荣科技(Silicon Motion)、LSI、联芸科技(Maxio)以及英韧科技(InnoGrit)。</li>
  <li data-pid="NI4H3YT3">常见DRAM制造商：三星电子(Samsung)、美光科技(Micron)、SK海力士半导体(SK Hynix)以及南亚科技(Nanya)。</li>
  <li data-pid="4iR2tu83">常见NAND制造商：三星电子(Samsung)、闪迪(Sandisk)、铠侠(Kioxia)、英特尔(Intel)、美光科技(Micron)以及长江存储(YMTC)。</li>
  <li data-pid="hmE_HDpl">注：LSI为Intel的高端消费级和企业级固态硬盘提供定制主控。联芸科技(Maxio)来自智微科技(JMcrion)。英韧科技(InnoGrit)来自美满电子(Marvell)。铠侠(Kioxia)由东芝存储器(Toshiba Memory)更名而来，全名铠侠株式会社。</li>
 </ul>
 <h3>9.有必要购买PCIe Gen4的NVMe固态硬盘吗？</h3>
 <p data-pid="i6RpDUaX">没有必要。如前文所述，即便是PCIe+NVMe固态硬盘和SATA+AHCI固态硬盘的日常使用体验差距也十分有限，因此建议普通消费者优先选择容量更大的固态硬盘。</p>
 <p data-pid="q20sbb7d">随着PCIe Gen4平台（AMD Matisse和Vermeer即Ryzen 3000和5000系列处理器（非APU）+X570/B550主板、AMD 3000系列线程撕裂者+TRX40主板、Intel Tiger Lake平台即第十一代移动酷睿处理器以及Intel Rocket Lake平台即第十一代桌面酷睿处理器）的逐渐普及，各大SSD制造商开始推出支持PCIe Gen4的NVMe固态硬盘（如三星980 PRO和西数SN850）。</p>
 <p data-pid="ZenDf6id">但目前PCIe Gen4的NVMe固态硬盘价格较高（500GB接近1000元，1TB接近2000元），因此对于普通消费者，建议优先选择性价比更高的PCIe Gen3的NVMe固态硬盘。如果想以较低成本尝鲜，也可以选择基于群联E16主控的二线品牌固态硬盘。</p>
 <h3>10.为什么不推荐二线厂商？</h3>
 <ol>
  <li data-pid="BaU6xaaO">二线厂商的产品几乎全为贴牌，无自主研发能力，多采用上游厂商提供的公版方案。</li>
  <li data-pid="YUHyqTyS">与一线厂商相比，二线厂商售后服务较差，配套软件和固件更新也不如一线厂商。</li>
  <li data-pid="vXHr71QD">二线厂商的产品受上游厂商影响较大，可能经常更换方案（缩水）。</li>
 </ol>
 <h3>11.如何理解固态硬盘的质保？</h3>
 <p data-pid="pWUZcrlZ">以三星970 EVO Plus 500GB为例，官方质保条件为：5年或300TBW，即自购买之日起，时间达5年或写入量达300TB（二者满足其一），保修终止。关于质保时间，一般按照发票上的购买日期计算，对于无发票的，则按出厂日期计算。</p>

  “MB/s” means 1,000,000 byte/sec. The result depends on Test File Size, Test File Position, Fragmentation, IDE(PATA)/SATA/RAID/SCSI controller and CPU speed etc… In a part of SSD, the result depends on test data(random, 0fill). 1GiB = 1024MiB = 1024x1024KiB = 1024x1024x1024B 1GB = 1000MB = 1000x1000KB = 1000x1000x1000B
 </blockquote>
 <p data-pid="mZ8pNAAj">CrystalDiskMark的逻辑跟AS SSD Benchmark不太一样，比如测试时先读后写，单位是千进制等。</p>
